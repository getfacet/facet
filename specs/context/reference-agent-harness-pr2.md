## Current Behavior Evidence

- `agent.ts:111-113` loads sink history, builds initial messages, and creates a
  `StageToolBuffer` directly inside the public factory.
- `agent.ts:115-136` runs the provider loop, executes each tool call, appends a
  `tool_result`, yields coalesced patch messages, and resets emitted patch ops.
- `agent.ts:137-164` catches provider/sink failures, drains unresolved buffered
  edits, surfaces final prose, and emits a fallback say if nothing happened.
- `prompt.ts:223-230` caps history only by entry count and includes full
  `session.stage` JSON in the final provider message.
- `provider.ts:89-103` has per-provider-call abort handling, but loop-level
  budgets beyond `maxSteps` live in `agent.ts`.
- `provider.ts:98-100` rejects HTTP failures with a provider/status message;
  `provider.ts:89-90` creates the per-call abort deadline. A harness retry
  classifier can inspect provider error status/name without seeing API keys.
- `agent-tools/buffer.ts:206-212` realigns local shadow after emitted patch
  batches through `foldPatchIntoStage`, which is the right primitive for PR2 to
  continue using.

## Required Hard Contract For PR2

- Stop reasons are a closed union:
  `provider_stop`, `max_steps`, `tool_call_limit`, `context_limit`,
  `provider_error`, `retry_exhausted`, `sink_error`, `unresolved_buffer`,
  `empty_turn`.
- Provider retry is allowed only when `provider.run(...)` rejects before any tool
  call from that provider step has executed. The harness must not retry after a
  patch or say was emitted for that step.
- Retryable provider failures are timeout/abort/network errors plus HTTP
  `408`, `409`, `425`, `429`, `500`, `502`, `503`, and `504`. HTTP `400`,
  `401`, `403`, `404`, and `422` are non-retryable.
- Default retry policy: `maxProviderRetries = 1`, `retryBackoffMs = 250`.
- No automatic cross-provider fallback. OpenAI to Anthropic or Anthropic to
  OpenAI switching remains an operator/deployer decision.
- Trace events are a closed union:
  `turn_start`, `context_compacted`, `provider_attempt`, `provider_retry`,
  `provider_step`, `tool_result`, `batch_yield`, `stop`, `turn_error`.
- Trace payloads must not include API keys, provider authorization headers,
  `visitorId`, full provider prompts, full stage JSON, or raw provider response
  bodies.
