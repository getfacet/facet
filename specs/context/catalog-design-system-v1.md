## Stage 0 Conclusion

GO for spec writing with these design constraints:

1. Implement v1 high-level bricks as a deliberate `@facet/core` protocol
   expansion, not as prompt-only naming.
2. Keep catalog as operator-provided UI vocabulary/policy data, not hosted
   platform authorization or business logic.
3. Keep theme recipes in validated theme documents and keep tree-authored style
   values token-only.
4. Keep table/chart data display-only and supplied by the agent; no client
   fetch, sorting, filtering, or backend binding in v1.
5. Keep overlay-class bricks out of v1.
6. Preserve fail-soft asset loading and optional renderer inputs for current
   consumers.
