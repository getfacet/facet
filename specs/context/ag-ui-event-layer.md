# Context Evidence: AG-UI Event Layer

> Generated by `/spec-bridge` Stage 0. Intake brief:
> `specs/feature-intake/ag-ui-event-layer.md`.

## Scope

- Feature slug: `ag-ui-event-layer`
- Goal: add an official AG-UI-facing event layer while preserving Facet's safe
  UI tree, server-authoritative stage, RFC 6902 patches, and shared
  fold/validation path.
- Implementation posture: AG-UI is an edge/event envelope. Facet's canonical
  stage model remains `FacetTree` + `JsonPatchOperation` + `foldPatchIntoStage`.

## Current Package and Invariant Evidence

- Facet invariant #1: agents emit declarative bricks, never raw HTML/JS. Evidence:
  `AGENTS.md:14`.
- Facet invariant #2: only RFC 6902 patches travel and the same pure patch fold
  runs on server and client. Evidence: `AGENTS.md:20`, `docs/ARCHITECTURE.md:201`,
  `packages/core/core/src/stage-fold.ts:33`.
- `@facet/core` is the dependency-free contract package. Evidence:
  `AGENTS.md:49`, `AGENTS.md:119`, `docs/ARCHITECTURE.md:421`.
- `@facet/server` is a reference transport, not a hardened public edge. Evidence:
  `AGENTS.md:51`, `docs/PACKAGE-BOUNDARIES.md:71`.
- `@facet/client` is the browser-side reference transport. Evidence:
  `AGENTS.md:52`, `docs/PACKAGE-BOUNDARIES.md:72`.
- AG-UI was already identified as a transport-only candidate, not a v0
  dependency. Evidence: `docs/ARCHITECTURE.md:407`, `docs/ARCHITECTURE.md:418`.

## Current Facet Event Path

- `ClientEvent` is the forward subset: `visit`, `message`, agent `tap`.
  Evidence: `packages/core/core/src/protocol.ts:99`.
- `ServerMessage` is native output: `patch`, `say`, server `reset`. Evidence:
  `packages/core/core/src/protocol.ts:120`.
- `FacetTransport` is the React hook's transport interface:
  `send`, `subscribe`, optional `record`. Evidence:
  `packages/core/core/src/protocol.ts:214`.
- External agent frames are native and core-owned:
  `AgentEventFrame` / `AgentControlFrame`. Evidence:
  `packages/core/core/src/protocol.ts:232`.
- `useFacet` consumes `FacetTransport`, folds native `patch`, clears chat on
  native `reset`, and appends native `say`. Evidence:
  `packages/core/react/src/useFacet.ts:68`, `packages/core/react/src/useFacet.ts:76`.
- `FacetRuntime` saves each folded batch before delivery. Evidence:
  `packages/core/runtime/src/runtime.ts:271`.
- `FacetRuntime.applyToSession` coalesces patch messages at the first patch slot
  and preserves `say` ordering. Evidence:
  `packages/core/runtime/src/runtime.ts:438`.
- Native browser transport hardcodes `/stream`, `/event`, and `/record`.
  Evidence: `packages/core/client/src/sse-transport.ts:31`,
  `packages/core/client/src/sse-transport.ts:54`,
  `packages/core/client/src/sse-transport.ts:106`.
- Native server routes hardcode `/stream`, `/event`, `/record`, and `/agent/*`.

## AG-UI Current External Evidence

- `@ag-ui/core` latest checked via npm: `0.0.57`, created
  `2025-04-30T11:36:28.217Z`, modified `2026-06-12T08:19:52.122Z`, dependency
  `zod`.
- `@ag-ui/client` latest checked via npm: `0.0.57`, created
  `2025-04-30T11:36:35.673Z`, modified `2026-06-12T08:19:34.182Z`, dependencies
  include `@ag-ui/core`, `@ag-ui/encoder`, `@ag-ui/proto`, `rxjs`,
  `fast-json-patch`, `uuid`, and `zod`.
- Official AG-UI event types in `@ag-ui/core@0.0.57` include
  `RUN_STARTED`, `RUN_FINISHED`, `RUN_ERROR`, `TEXT_MESSAGE_START`,
  `TEXT_MESSAGE_CONTENT`, `TEXT_MESSAGE_END`, `STATE_SNAPSHOT`,
  `STATE_DELTA`, `CUSTOM`, `RAW`, tool, activity, reasoning, and capability
  types.
- `STATE_DELTA` carries an array of JSON Patch operations and `STATE_SNAPSHOT`

- `RunAgentInput` includes `threadId`, `runId`, `state`, `messages`, `tools`,
  `context`, `forwardedProps`, and optional `resume`. `forwardedProps` is the
  appropriate extension point for Facet-specific `visitor/event/record`
  envelopes, but it is untrusted and must be narrowed.

## Risk Register From Stage 0

### Invariant Risks

- `RISK-INV-1`: AG-UI `state` must not become a generic renderer write path.
  Facet accepts only reserved Facet stage state and still folds/validates.
- `RISK-INV-2`: `STATE_SNAPSHOT` must be server-authoritative only. Client or
  agent-supplied `RunAgentInput.state` must not seed or overwrite `StageStore`.
- `RISK-INV-3`: local `navigate` / `toggle` remain browser view-state and log-only
  records. AG-UI custom/record mapping must preserve the `/event` vs `/record`
  split.
- `RISK-INV-4`: malformed AG-UI event bodies must fail closed before
  runtime/renderer. Existing native validators cap and reject malformed bodies.
- `RISK-INV-5`: text/state ordering can drift if AG-UI streaming is mapped too
  loosely. Mapping must preserve runtime batch order.
- `RISK-INV-6`: external AG-UI compatibility must not become direct
  browser-to-agent stage authority. Server-mediated AG-UI is required.
- `RISK-INV-7`: AG-UI tool/custom capability must not add backend/domain work to
  Facet. Tool events may be observable protocol data only.

### Public API Risks

- `RISK-API-1`: native protocol types are public from `@facet/core`; replacing
  them directly would be a contract break and would pull AG-UI toward core.
- `RISK-API-2`: `useFacet` consumes native `FacetTransport`; AG-UI must adapt to
  that interface or `useFacet` becomes breaking.
- `RISK-API-3`: `@facet/client` exports native transports; public posture needs
  wrapper/demotion docs.
- `RISK-API-4`: `SseTransport` hardcodes native routes; it cannot silently become
  AG-UI without a breaking semantic change.
- `RISK-API-5`: `createFacetServer` exposes native routes; adding AG-UI to the
  server should be additive or separated to avoid dependency cycles.
- `RISK-API-6`: external-agent API is native dial-out and may not match AG-UI's
  HTTP agent model.
- `RISK-API-7`: agent authoring remains native `ClientEvent` + `Stage` +
  `ServerMessage[]`; AG-UI should stay at the edge to avoid churn.
- `RISK-API-8`: quickstart/playground currently instantiate native transports.
  The new public AG-UI path must be tested and docs must mark native as fallback.
- `RISK-API-9`: persistence stores native `CollectedEvent` and `ServerMessage[]`.
  Storing raw AG-UI events would be a durable schema migration.
- `RISK-API-10`: docs currently describe native protocol as the reference surface;
  AG-UI promotion requires docs changes.

### Package Risks

- `RISK-PKG-1`: `@ag-ui/*` must not enter `@facet/core`.
- `RISK-PKG-2`: a new package must fit `packages/*/*`, path aliases, exports,
  and build metadata.
- `RISK-PKG-3`: browser-safe entrypoints must not import Node-only server code.
- `RISK-PKG-4`: central adapter must not create dependency cycles with client,
  server, or agent-client packages.
- `RISK-PKG-5`: dev `src` exports and published `dist` exports must mirror
  subpaths.
- `RISK-PKG-6`: quickstart can silently remain native unless docs/tests identify
  the native path as fallback and the AG-UI adapter path is covered.
- `RISK-PKG-7`: external agent dial-out may need a future dedicated
  `@facet/ag-ui/agent` adapter; do not collapse this into browser/server work.

## GO / NO-GO

GO with constraints:

- Add `packages/extensions/ag-ui` as `@facet/ag-ui`.
- Use official `@ag-ui/core` and `@ag-ui/client` in the adapter package.
- Keep `@facet/core`, `@facet/runtime`, and `@facet/react` canonical and AG-UI
  unaware.
- Provide browser-safe index/transport exports and a Node-only `./server`
  subpath.
- Keep native reference transport as an internal/reference fallback for this
  change; document AG-UI as the new external integration path.
