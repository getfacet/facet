## Independent risk probes

### INV lens

#### RISK-INV-001 — A fixed-choice reference can become browser/domain filtering

- Evidence: old filter dispatch is `brick-renderer-inputs.tsx:113-130,167-219`;
  architecture explicitly rejects renderer-authored queries
  (`docs/ARCHITECTURE.md:393-430`).
- Required control: reference is only pressable box + label text + local
  navigate/active screen. Recursively assert absence of input/filter/onChange,
  Apply/agent/collect, fetch/query/where/predicate/formula/resolver/dataSource.
  Data/tree bytes remain unchanged on click.

#### RISK-INV-002 — References can become a third tier or carry dangling screens

- Evidence: compositions deliberately have no screens map
  (`composition-validation.ts:54-64`); read-only lookup returns the same shadow
  and no stage outputs (`agent-tools/src/executor-assets.ts:5-48`).
- Required control: lookup stays exact-name/read-only; metadata tells the agent
  to map illustrative names; structural tests require `onPress.to ===
  active.screen`; real-tree tests provide matching screens; dangling targets
  no-op/plain-degrade without action/record/throw.

#### RISK-INV-003 — Stale retired containers can leak descendants

- Evidence: form is currently a container (`nodes.ts:417-427`) and motion recurses
  into containers (`renderer-motion.ts:184-202`).
- Required control: matrix for all seven retired names across core roots/children,
  screens, React raw siblings/subtrees/motion, set/append tools, repeated calls,
  and prototype names. Stale form descendants are not rendered or motion-tracked.

#### RISK-INV-004 — Selected styling can become a raw-style/special mechanism

- Evidence: `activeVariant` is already a recipe selector and `activeStyle` is
  token-sanitized (`nodes.ts:197-215`, `primitive-node-validation.ts:337-396`);
  view predicates are closed/total (`view.ts:160-245`).
- Required control: ordinary `recipes.box.selected` (and text only if used), no
  selected-specific resolver/part/renderer branch, hostile raw styles dropped,
  unknown recipes plain-degrade.

#### RISK-INV-005 — Survivor reclassification can silently change behavior

- Evidence: normalizers live in data/feedback validators; resolver/content hooks
  live in `brick-registry.ts:201-230,249-260`; React registry maps all six at
  `brick-render-registry.ts:69-77`.
- Required control: pre-base normalized JSON plus exact ordered issues and SSR
  markup goldens; cover caps/bindings/dangling/sort/progress/loading/malformed
  cases. Structural roster tests supply RED; preservation goldens must remain
  byte-identical.

#### RISK-INV-006 — “No agent event” can be mistaken for no local record

- Evidence: `StageRenderer.tsx:175-236` records valid local actions while sending
  no `onAction`; current tests pin `onAction=0`, `onRecord=1`.
- Required control: tabs/nav/filter click changes latest screen locally, keeps
  stage/data/patches unchanged, emits zero agent actions and exactly one ordered
  record per valid click; active evaluation emits none; dangling target emits
  neither; record failure does not unwind state.

#### RISK-INV-007 — Backend/query behavior can leak through another name

- Evidence: architecture confines backend calls to agents and composition
  preflight already rejects known backend-like fields.
- Required control: fetch spy remains zero; recursive asset guards plus
  addition-only source scan classify any fetch/queryExpr/dataSource/resolver/
  where/predicate/formula/onChange addition.

Invariant 5 found no distinct new risk if references remain ordinary flow boxes;
tests still reject overlay/position/z keys and exercise long labels in narrow
containers.
