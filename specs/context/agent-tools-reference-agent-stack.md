## Risk Probe: INV

### RISK-INV-1 — Agent tools must not widen Facet's declarative stage boundary

Evidence:
- Facet's load-bearing rule is declarative bricks and token styles:
  `AGENTS.md:15-24`.
- `validateTree` is the fail-safe boundary for arbitrary LLM JSON and strips
  unknown node types / invalid tokens without throwing:
  `packages/core/src/validate.ts:48-57`.

Resolution requirement:
- `@facet/agent-tools` only emits existing `ServerMessage` patch/say messages
  over existing `box` / `text` / `media` / `field` nodes and token styles.
- No raw HTML, raw JS, raw CSS, arbitrary URL fetch, pixel positioning, new
  brick, or new token vocabulary is introduced.

### RISK-INV-2 — Tool-local stage shadow can drift from runtime if it does not use canonical fold

Evidence:
- `foldPatchIntoStage` is documented as the single pure fold used by both server
  and client and never throws: `packages/core/src/stage-fold.ts:27-53`.
- The current agent loop executes multiple provider steps before and after
  `yield`, but generator code does not receive the runtime's applied stage back:
  `packages/reference-agent/src/agent.ts:539-580`.

Resolution requirement:
- `@facet/agent-tools` maintains a local shadow by folding only emitted patch
  messages through `foldPatchIntoStage`.
- The executor returns compact applied-stage summaries from that folded shadow.
- Runtime remains authoritative after yielded batches; PR2 context assembly must
  seed each turn from `session.stage`.

### RISK-INV-3 — Fail-safe observations must replace normal throws for bad model input

Evidence:
- Current `executeTool` promises never to throw and turns bad args into
  observations: `packages/reference-agent/src/agent.ts:349-361`.
- Existing tests assert bad tool args are fed back and recover:
  `packages/reference-agent/src/agent.test.ts:697-722`.

Resolution requirement:
- `@facet/agent-tools` executor returns typed `error` observations and emits no
  patch on invalid mutation inputs.
- Unexpected internal exceptions may still be surfaced as executor failure in
  tests, but model-authored invalid calls use normal observation control flow.

### RISK-INV-4 — Closure buffering and patch caps are easy to regress when moved

Evidence:
- Current closure buffer is responsible for avoiding partial dangling-child
  batches: `packages/reference-agent/src/agent.ts:219-321`.
- `use_stamp` checks the patch cap before emitting expansion:
  `packages/reference-agent/src/agent.ts:465-470`.
- `MAX_PATCH_OPS` fold cap exists in core:
  `packages/core/src/stage-fold.ts:69-84`.

Resolution requirement:
- PR1 must port closure buffering and patch cap behavior into
  `@facet/agent-tools` with focused tests before reference-agent consumes it.

### RISK-INV-5 — `@facet/agent-tools` must not become a backend/domain tool package

Evidence:
- Facet boundary says domain/backend work is out of scope:
  `docs/ARCHITECTURE.md:335-343`.

Resolution requirement:
- `@facet/agent-tools` contains only stage mutation, say, inspection, and stage
  shadow helpers. It does not include provider calls, fetches, database adapters,
  purchases, business API tools, or quickstart assets policy.

## Risk Probe: API

### RISK-API-1 — Moving `ToolSpec` / `ToolCall` can break existing reference-agent consumers

Evidence:
- `ToolSpec`, `ToolCall`, `ProviderStep`, and `ProviderTurn` are public through
  `packages/reference-agent/src/provider.ts:20-65` and `src/index.ts:3`.
- `packages/reference-agent/src/index.test.ts:47-80` pins the public type surface.

Resolution requirement:
- `@facet/agent-tools` can own the canonical `ToolSpec` / `ToolCall` types, but
  `@facet/reference-agent` must re-export compatibility type aliases.

### RISK-API-2 — Quickstart depends on reference-agent compatibility re-exports

Evidence:
- `packages/quickstart/src/index.ts:1-6` re-exports reference-agent.
- `packages/quickstart/src/provider.ts:1-19`, `prompt.ts:1-9`, and `stub.ts:1`
  re-export specific reference-agent APIs.

Resolution requirement:
- Reference-agent public names `createQuickstartAgent`, `createReferenceAgent`,
  `QuickstartAgentOptions`, `ReferenceAgentOptions`, `QuickstartProvider`,
  `ReferenceProvider`, `TOOLS`, `buildSystem`, `buildInitialMessages`,
  `createStubAgent`, and provider helpers remain importable.

### RISK-API-3 — New `@facet/agent-tools` must be usable without reference-agent

Evidence:
- Intake goal requires external agent authors to attach Facet tools to their own
  LLM loop without importing reference-agent.
- Current tool schemas and executor are private to reference-agent:
  `packages/reference-agent/src/prompt.ts:134-215`,
  `packages/reference-agent/src/agent.ts:343-525`.

Resolution requirement:
- Add an import/type test or example that imports from `@facet/agent-tools` only,
  executes a valid and invalid tool call, and does not import
  `@facet/reference-agent`.

### RISK-API-4 — Physical directory migration should not rename npm package names

Evidence:
- Package names are stable in each manifest, e.g.
  `packages/core/package.json:2`, `packages/reference-agent/package.json:2`,
  and `packages/quickstart/package.json:2`.

Resolution requirement:
- Directory paths change, but `name`, public exports, and package import
  specifiers stay unchanged.
