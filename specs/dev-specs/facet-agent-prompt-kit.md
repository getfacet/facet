## Overview

- Feature slug: `facet-agent-prompt-kit`
- Intake brief: `specs/feature-intake/facet-agent-prompt-kit.md`
- Context evidence: `specs/context/facet-agent-prompt-kit.md`
- Affected `@facet/*` packages: `@facet/agent-tools`, `@facet/reference-agent`
- One-paragraph technical approach: add a provider-neutral prompt-kit module to
  `@facet/agent-tools` and export it from the root barrel. The kit owns reusable
  Facet LLM guidance: role, `STAGE_SPEC` inclusion, page experience principles,
  state-editing strategy, tool playbook with structured tool-result recovery,
  asset/privacy rules, and final page-brief composition. Keep event/history/stage
  context assembly, `DEFAULT_GUIDE`, providers, and the harness inside
  `@facet/reference-agent`. Refactor `@facet/reference-agent buildSystem()` into
  a compatibility wrapper around the new helper while keeping `TOOLS` identity
  and public exports intact.

## Fail-safe & Boundary Checklist

- Malformed / empty / null input: the public builder requires `pageBrief: string`
  at type level; empty string still builds a prompt ending in `PAGE BRIEF`.
  `assets` is optional; missing/empty arrays omit injected asset sections. Covered
  by WU-1 prompt-kit tests.
- Deeply nested / cyclic tree: out of scope for prompt assembly; existing stage
  validation and stage-shadow tooling remain unchanged. Covered by WU-2
  no-regression test target `packages/core/core/src/validate.test.ts`, including
  cycle and pathologically deep tree cases.
- Offline agent / rapid events / racing agent turn: out of scope for prompt
  assembly; no runtime/harness event ordering changes. Covered by WU-2
  no-regression test targets `packages/agent-stack/reference-agent/src/harness/loop.test.ts`
  and `packages/core/runtime/src/runtime.test.ts`.
- Sensitive values: tests must prove no theme CSS values, stamp node ids/JSON,
  provider-key-looking values, API-key text, or visitor id text are introduced by
  the asset sections. Covered by WU-1/WU-2 prompt tests.
- Tool failure observations: prompt tests must preserve guidance for
  `applied_visible`, `applied_not_visible`, `applied_with_warnings`, `pending`,
  `rejected`, and `no_stage_change`. Covered by WU-1/WU-2 prompt tests.
