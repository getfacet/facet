## Locked Design

1. Final native roster: `box`, `text`, `media`, `input`, `richtext`, `table`,
   `chart`, `list`, `keyValue`, `progress`, `loading`. Public names are
   `BRICK_TYPES`, `BrickType`, and `FacetNode`.
2. `ContainerNode`/`isContainer` remain a box-only capability, not a tier.
3. Survivor public interfaces (`TableNode`, `ChartNode`, `ListNode`,
   `KeyValueNode`, `ProgressNode`, `LoadingNode`, supporting types) remain root
   exports. `component-nodes.ts` is deleted.
4. Button/form/filter/metric/tabs/nav are optional native reference examples;
   `stat` is deleted without an alias or separate reference.
5. CTA/submit/category/tab/nav controls are pressable boxes plus label text. A
   form is a box with inputs and a pressable submit box. A metric is label/value
   text; value may bind via `from`/`column`/`row`.
6. Tabs/nav/fixed-choice filter use local `navigate`, identical
   `onPress.to`/`active.screen`, and `activeVariant: "selected"`. The agent maps
   illustrative targets to preauthored screen names.
7. A valid local click changes only browser view state, sends zero agent action,
   emits exactly one ordered record, and emits no stage patch. A dangling target
   emits neither action nor record. A synchronous record callback throw cannot
   unwind the local view change; PR-6 does not widen the callback to Promise.
8. The fixed filter demonstrates only bounded preauthored-screen selection. No
   Apply, `input.onChange`, query/filter language, fetch, resolver, formula, or
   client data mutation is added. Open-ended work remains agent/backend-owned.
9. Selected styling is ordinary token-validated `recipes.box.selected` (and
   text selected only if used). No selected-specific renderer or box ability.
10. Public hard rename, without aliases:
    `ComponentRecipePart`→`BrickRecipePart`,
    `ComponentRecipeParts`→`BrickRecipeParts`,
    `ComponentRecipe`→`BrickRecipe`,
    `ComponentRecipes`→`BrickRecipes`,
    `RECIPE_COMPONENTS`→`RECIPE_BRICKS`,
    `RecipeComponentName`→`RecipeBrickName`. `MAX_TABS_ITEMS` is removed.
11. Catalog retains bricks, compositions, theme, and edit policy. It removes
    `CatalogComponent`, `components`, `primitiveFallback`,
    `CatalogUsageOrder`, and `policy.order`. Absent bricks default to all 11;
    explicitly invalid/all-invalid provided bricks remain `[]`.
12. Boundary outcomes differ intentionally: core prunes retired nodes, tools
    clean-reject with identical shadow/zero patch, and React skips the complete
    stale raw subtree while rendering valid siblings.

## Fail-safe & Boundary Checklist

- WU-04 owns explicit null, empty, malformed, deep, cyclic, retired root/child/
  screen, repeated invalid, and prototype-name core cases in `validate.test.ts`.
- WU-12 proves all seven stale raw types blank-degrade at root/child/screen and a
  stale form's descendants never render. WU-13 separately owns stale-subtree
  motion evidence.
- WU-13 renders tab/nav/filter references with agent channel absent and
  unavailable: local latest-screen and selected look work; action/patch/tree/
  data remain unchanged; the next view snapshot is current.
- WU-13 fires rapid multi-click sequences and asserts latest-screen wins, exact
  ordered one-record-per-valid-click, active evaluation emits no record,
  dangling emits neither, and a synchronous record throw cannot unwind state.
- WU-14 covers set/append for all seven retired names, repeated calls, and
  constructor/toString/other prototype names: clean error, same shadow, zero
  patch.
- WU-03/WU-05/WU-10–13 retain normalized JSON/ordered issues, binding, caps,
  sort, progress bounds, loading motion, malformed behavior, and SSR output for
  the six survivors.
- WU-01 uses a per-pattern matrix: tabs/nav/filter are box+text-only and reject
  input/Apply/onChange/agent/collect/fetch/query/where/predicate/formula/
  resolver/dataSource; form positively requires inputs plus one exact closed
  agent action with `collect` naming its container. Every pattern rejects raw
  HTML/JS/CSS/pixels and position/overlay/z-index fields.
