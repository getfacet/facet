- One-paragraph technical approach:
  PR2 keeps the public `@facet/reference-agent` package surface compatible while
  replacing the monolithic `agent.ts` loop with a named harness. Provider
  adapters move behind `provider/`, prompt/context assembly moves behind
  `prompt/`, and turn execution moves behind `harness/` modules for transcript,
  context, compaction, budget, loop, and trace. The harness consumes
  `@facet/agent-tools` for all stage-tool execution and local shadow folding.
  Core/runtime/protocol/renderer stay untouched.

## Reference Harness Contract

PR2 is only complete if these contracts are implemented exactly and pinned by
tests. They are intentionally concrete so this does not land as a partial
refactor.

### Module Contract

```txt
packages/agent-stack/reference-agent/src/
  index.ts
  agent.ts
  provider.ts
  prompt.ts
  stub.ts

  provider/
    types.ts
    openai.ts
    anthropic.ts
    resolve.ts

  prompt/
    system.ts
    messages.ts
    stage-summary.ts

  harness/
    loop.ts
    transcript.ts
    context.ts
    compaction.ts
    budget.ts
    trace.ts
```

- Top-level `provider.ts` and `prompt.ts` remain compatibility barrels.
- `agent.ts` becomes the public factory/configuration layer. It must not own
  provider-step loop logic after WU-7.
- `harness/loop.ts` owns turn execution and is the only place that calls
  `provider.run(...)`.
- Stage tools are executed only through `@facet/agent-tools`.

### Budget Profile Contract

The default profile is a **hosted quickstart safety default**, not a claim that
all production agents should stop at these limits. The reference agent exposes
named profiles plus explicit overrides so deployers can choose the cost/latency
envelope they want.

`ReferenceAgentBudgetPreset` is a closed union:

```ts
type ReferenceAgentBudgetPreset = "quickstart" | "hosted" | "local-dev";
```

| Field | `quickstart` default | `hosted` preset | `local-dev` preset | Meaning |
|---|---:|---:|---:|---|
| `maxSteps` | `50` | `120` | `240` | Provider steps per visitor turn. |
| `maxToolCallsPerStep` | `16` | `32` | `64` | Maximum tool calls accepted from one provider step. |
| `maxContextChars` | `96000` | `160000` | `240000` | Estimated `system + transcript` character cap before provider call. |
| `maxHistoryTurns` | `20` | `40` | `80` | Stored sink turns considered before char compaction. |
| `maxHistoryChars` | `24000` | `48000` | `96000` | Character cap for replayed history after deterministic compaction. |
| `maxStageJsonChars` | `48000` | `80000` | `120000` | Full current stage JSON is included only at or below this cap. |
| `maxStageSummaryNodes` | `80` | `160` | `320` | Maximum node summaries included when full stage JSON is too large. |
| `maxObservationChars` | `4000` | `8000` | `12000` | Per-tool observation cap before transcript insertion. |
| `maxFinalTextChars` | `4000` | `8000` | `12000` | Clean provider prose cap before surfacing as chat. |
| `maxProviderRetries` | `1` | `2` | `2` | Retries per provider step after a retryable provider failure. |
| `retryBackoffMs` | `250` | `500` | `0` | Delay before retry attempt. Tests may set `0`. |

Preset intent:

- `quickstart`: default for public demos and npx-first hosted entrypoints. It is
  intentionally conservative to avoid runaway cost and latency.
- `hosted`: opt-in production profile for an operator serving real visitors who
  accepts a larger per-turn budget.
- `local-dev`: opt-in local experimentation profile for a developer using their
  own API key. It is generous but still bounded; no built-in profile is
  unbounded.

Compatibility precedence:

- `budgetPreset` defaults to `"quickstart"`.
- `budget` overrides the selected preset field-by-field.
- Existing `maxSteps` option remains supported. If `budget.maxSteps` is present,
  it wins; otherwise `options.maxSteps` wins; otherwise the selected preset
  value wins.
- Existing `historyTurns` option remains supported. If
  `budget.maxHistoryTurns` is present, it wins; otherwise
  `options.historyTurns` wins; otherwise the selected preset value wins.
- All new budget fields are optional and must not be required by existing
  callers.
- `Infinity`, `Number.MAX_SAFE_INTEGER`, negative values, `NaN`, and fractional
  values are rejected or normalized to safe integers by `normalizeBudget`.

### Stop Reason Contract

`ReferenceAgentStopReason` is a closed string union:

```ts
type ReferenceAgentStopReason =
  | "provider_stop"
  | "max_steps"
  | "tool_call_limit"
  | "context_limit"
  | "provider_error"
  | "retry_exhausted"
  | "sink_error"
  | "unresolved_buffer"
  | "empty_turn";
```

Meanings:

- `provider_stop`: provider returned no tool calls and the turn ended cleanly.
- `max_steps`: the loop reached `maxSteps` while the provider kept requesting
  tools.
- `tool_call_limit`: one provider step returned more than
  `maxToolCallsPerStep`; no tool from that oversized step is executed.
- `context_limit`: deterministic compaction still cannot bring context under
  `maxContextChars`; no provider call is made for that turn.
- `provider_error`: non-retryable provider failure.
- `retry_exhausted`: retryable provider failure remained after configured
  retries.
- `sink_error`: sink history read failed before provider context could be built.
- `unresolved_buffer`: turn ended with unresolved `@facet/agent-tools` buffered
  nodes.
- `empty_turn`: no mutation, no say, and no clean final prose were produced.

### Retry And Fallback Contract

- Retry is allowed only for a failed `provider.run(...)` call before any tool
  from that provider step executes.
- Retrying after any stage patch or say from that provider step is forbidden.
- Retryable failures:
  - timeout / abort errors,
  - network `TypeError` from `fetch`,
  - HTTP `408`, `409`, `425`, `429`, `500`, `502`, `503`, `504`.
- Non-retryable failures:
  - HTTP `400`, `401`, `403`, `404`, `422`,
  - malformed provider response,
  - unknown provider/flag resolution errors.
- No automatic cross-provider fallback. Provider switching is a deployer policy,
  not a harness behavior.
- If prior batches were already yielded in the turn, provider failure keeps them.
- Fallback say is emitted only when the turn otherwise produced no mutation,
  no `say`, and no clean final prose, or when unresolved buffered edits require
  explicitly telling the user the requested update could not be completed.
- Fallback say is emitted at most once per turn.

### Context And Compaction Contract

- The current visitor event is never dropped.
- Full current stage JSON is included only when it is at or below
  `maxStageJsonChars` and the full context remains at or below
  `maxContextChars`.
- If the stage is too large, context includes:
  - root id, node count, screen count, entry, theme,
  - up to `maxStageSummaryNodes` deterministic node summaries sorted by id,
  - a clear instruction to use `inspect_stage` / `inspect_node` for details.
- History compaction is deterministic and never calls a model. It keeps the
  newest entries first, emits a bounded compaction note with dropped counts, and
  never includes raw patch arrays.
- If compaction still cannot fit the context, the loop stops with
  `context_limit` before making a provider call.

### Observation Contract

- Every executed tool call appends exactly one `tool_result` transcript entry
  before the next provider call.
- Observation text longer than `maxObservationChars` is truncated to fit the cap
  and ends with a marker of the form
  `[truncated: <N> chars omitted]`.
- Truncation is recorded as a trace event and does not itself stop the turn.

### Trace Contract

`ReferenceAgentTraceEvent["type"]` is a closed union:

```ts
type ReferenceAgentTraceEventType =
  | "turn_start"
  | "context_compacted"
  | "provider_attempt"
  | "provider_retry"
  | "provider_step"
  | "tool_result"
  | "batch_yield"
  | "stop"
  | "turn_error";
```

- Trace is optional: `trace?: (event: ReferenceAgentTraceEvent) => void |
  Promise<void>`.
- Trace callback failures are ignored and must never alter the agent turn.
- Trace payloads are bounded metadata only. They must not include API keys,
  provider authorization headers, `visitorId`, full provider prompts, full stage
  JSON, or raw provider response bodies.
- `stop` trace includes the final `ReferenceAgentStopReason`.

## Fail-safe & Boundary Checklist

- Malformed / empty / null input:
  - Provider tool calls still run through `@facet/agent-tools`, producing error
    observations and no patch on invalid input.
  - Corrupt stored history events remain printable as `(unknown event)`.
- Deeply nested / cyclic / huge stage:
  - Context assembly does not blindly include full JSON when it exceeds
    `maxStageJsonChars`; it emits a bounded summary and tells the model to use
    `inspect_stage` / `inspect_node`.
  - Stage mutations still fold through `@facet/agent-tools` / `@facet/core`.
- Offline agent / provider failure:
  - Retryable provider rejection retries only before any tool execution for that
    provider step. Non-retryable or exhausted failures stop the loop, preserve
    already yielded batches, and emit fallback only under the fallback contract.
- Rapid events / racing agent turn:
  - Runtime per-visitor serialization remains authoritative; the harness does
    not add cross-turn mutable global state.
- Oversized observations:
  - Tool observations are truncated before transcript insertion with an explicit
    marker, and stop/budget trace records the truncation.
- Runaway loops:
  - `maxSteps`, `maxToolCallsPerStep`, provider retry exhaustion, and
    context/observation budgets produce stable stop reasons covered by tests.
