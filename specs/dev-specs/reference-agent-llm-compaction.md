## Overview

- Feature slug: `reference-agent-llm-compaction`
- Intake brief: `specs/feature-intake/reference-agent-llm-compaction.md`
- Context evidence: `specs/context/reference-agent-llm-compaction.md`
- Affected `@facet/*` packages:
  - `@facet/runtime` (new brain-agnostic `SummaryStore` interface + memory/file refs)
  - `@facet/store-postgres` (`PostgresSummaryStore`)
  - `@facet/reference-agent` (token accounting, summarizer, assembly, loop, wiring)
  - `@facet/quickstart` (default wiring)
- One-paragraph technical approach:
  Add a brain-agnostic, opaque-payload `SummaryStore` to `@facet/runtime`
  (memory + file refs) and `@facet/store-postgres`, all keyed by
  `agentId`+`visitorId` with a monotonic `coveredThrough` guard. Teach both
  provider adapters to surface `ProviderStep.usage` and give the Anthropic
  adapter `cache_control` on its stable prefix. Extend `ReferenceAgentBudget`
  additively with a token-based budget model (char fields become legacy
  aliases) plus a calibrated chars-per-token estimator. Add a
  reference-agent-owned summary schema + `validateSummary` + a forced-tool-call
  summarizer that redacts its own output. Wire two compaction surfaces: a
  background, per-`(agentId,visitorId)`-serialized post-turn cross-turn
  compaction that persists rolling generations, and a synchronous in-turn
  tool-transcript compaction at the previous `context_limit` boundary. Every
  failure degrades to the existing deterministic truncation
  (`compactHistoryMessages`), keeping the stub path byte-for-byte deterministic.

## Fail-safe & Boundary Checklist

- **Malformed / empty / null summary payload:** `validateSummary` (reference-agent)
  returns undefined â†’ assembly proceeds deterministically; store returns the
  opaque payload verbatim and never interprets it.
- **Summarizer throw / timeout / invalid tool args (Ã—2):** retry once, then skip;
  previous generation stays; `compaction_failed` trace; turn completes normally.
- **Store read failure:** assemble without summary (deterministic) + trace.
- **Store write failure:** drop + trace; never blocks the turn.
- **`coveredThrough` > sink length (external wipe/reset):** discard summary,
  assemble deterministically, next compaction rebuilds at generation 1 (DC-015).
- **Stale summary (marker lags sink by K):** top-up replays the K post-marker
  turns verbatim; deterministic compaction is the final size guard (DC-014).
- **In-turn boundary:** never split a `assistant_tools`+`tool_result` pair;
  replace only contiguous whole step groups; keep last `minRecentStepsVerbatim`;
  refresh CURRENT STAGE from `buffer.shadow` with the same deterministic
  formatter. If still over budget â†’ existing `context_limit` stop remains.
- **No history (`NullSink`/`ForwardSink`) or no `SummaryStore`:** cross-turn path
  is a no-op; zero summarizer calls (DC-010).
- **Provider response without usage:** keep calibrated estimate only (clamped
  1.5â€“6 chars/token, conservative default first call); trace notes estimate-only.
- **Concurrent turns for the same visitor:** serialized on the reference-agent
  lane; monotonic `coveredThrough` makes a stale writer lose; process restart
  loses only the in-flight call (acceptable, rebuilt next trigger).
- **Cache-control ignored by API:** harmless additive body fields; OpenAI body
  unchanged.
- **Deterministic/stub path:** summarizer is injectable and never constructed on
  the stub path; Tier-1 stays byte-for-byte deterministic (DC-007).
