## Fail-safe & Boundary Checklist

- Malformed / empty / null input: provider tests cover malformed provider bodies, HTTP errors, timeouts, and parsed tool-call argument failures; agent tests cover bad tool args becoming `error:` observations and no-key CLI stays sanitized.
- Deeply nested / cyclic tree: no core tree validator changes are planned; WU-3 reference-agent tests must include fail-safe `render_page` empty/non-renderable plus deep/cyclic-object input cases so bad model/tool output cannot wipe the stage or throw.
- Offline agent / rapid events / racing agent turn: provider failure continues to degrade to a single apology without throwing; runtime/session serialization remains unchanged; WU-6 keeps the existing quickstart rapid double-message ordering E2E green, and final gates keep runtime/server serialization suites green.
