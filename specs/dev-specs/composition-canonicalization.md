## Overview

- Feature slug: `composition-canonicalization`
- Intake brief: `specs/feature-intake/composition-canonicalization.md`
- Context evidence: `specs/context/composition-canonicalization.md`
- Affected packages: `@facet/core`, `@facet/assets`, `@facet/runtime`,
  `@facet/agent-tools`, `@facet/reference-agent`, `@facet/quickstart`,
  `@facet/agent`, `@facet/store-postgres`
- Technical approach: replace the unreleased Stamp compatibility family with one
  canonical composition contract from validation through catalog, assets,
  server-side expansion, agent tools, SDK, file storage, and Postgres. Merge the
  two existing validation safety layers, collapse the runtime's three raw inputs
  and two outputs to one `compositions` path, migrate every serialized tool/error
  value together, and delete all old aliases/extensions. React and wire protocol
  remain unchanged because visitors still receive only validated nodes and RFC
  6902 patches.

## Fail-safe & Boundary Checklist

- Malformed/empty/null/hostile composition: `validateComposition` returns no
  composition plus bounded sanitized issues, never throws.
- Deep/cyclic/shared/dangling graph: same topology sanitizer as `validateTree`;
  invalid refs are pruned and depth/cycle caps remain.
- Oversized document/file/output: 1024 raw nodes reject the whole composition;
  reachable output is at most `MAX_PATCH_OPS - 1` (1023); issue output is at
  most `MAX_ISSUES` (64) plus one suppression tail; more than 5000 existing ids
  reject expansion; all mint attempts share one 4096-attempt budget. FileAssets
  stops at directory entry 4097, closes enumeration, returns empty raw documents
  with a bounded issue, and opens/decodes/parses zero files so boot falls back
  safely. At or below 4096 entries it sorts bounded names, opens at most the
  first 1024 theme and first 1024 composition files, and reads at most 1048577
  bytes each. Exactly 1048576 bytes may decode/parse; byte 1048577 rejects before
  decode/parse even when initial size is 1 MiB and the file grows during read.
- Throwing getter/proxy/mint/iterable: caught at validator/expander/asset seams.
  Exception detail access is itself guarded; C0/DEL/C1 are removed, the retained
  detail is at most 256 characters, and a throwing `message` getter cannot expose
  the thrown sentinel. Core/tool/buffer/SDK tests assert zero nodes/ops/messages,
  identical shadow/buffer/known-id state, and a later valid expansion still works.
- Unknown/disallowed composition or non-container parent: structured
  `invalid_composition`/`invalid_parent`, no patches.
- Offline agent: WU-11's deterministic quickstart E2E proves no response leaves
  the stage coherent and server healthy without throwing.
- Rapid/racing turns: WU-10 mutates the source after agent creation and drives
  rapid sequential turns, proving one immutable snapshot and ordered closed
  batches.
- Browser boundary: no composition JSON/default library is shipped in shell,
  protocol, renderer, client, AG-UI, or agent-client code.
