## Overview
- Feature slug: `box-concern-packs`
- Intake brief: `specs/feature-intake/box-concern-packs.md`
- Context evidence: `specs/context/box-concern-packs.md`
- Affected `@facet/*` packages: `@facet/core` (source-shape only), plus repo docs
  (`docs/ARCHITECTURE.md`, `AGENTS.md`).
- One-paragraph technical approach: This is PR-2.5 of the node-model restructure —
  a **pure type-space refactor + a governing-rule doc**, behavior-preserving with
  **zero runtime change** (packs are TypeScript interfaces; they erase at compile).
  In `packages/core/core/src/nodes.ts` we extract `box`'s remaining loose behavior
  fields into two module-private field packs — `Pressable { onPress?; onHold? }`
  and `Layered { backdrop? }` — mirroring PR-1.5's `BaseNode`/`Styleable`/
  `ActiveLook`/`ContainerFields`/`DataBound` precedent, and recompose `BoxNode` via
  `interface BoxNode extends BaseNode, Styleable<BoxStyle>, ActiveLook<BoxStyle>,
  ContainerFields, Pressable, Layered`. The `type: "box"` discriminant and
  `hidden?: boolean` stay **direct own members** of `BoxNode` (PR-1.5 lesson +
  visibility ≠ layering; see RISK-HIDDEN). `BoxNode`'s resolved key set is
  byte-identical, so the existing PR-1.5 field-set/exhaustiveness guards stay green
  and no assertion is rewritten. We then write the two-axis **brick-vs-field growth
  rule** (plus one fuzzy-middle line) once in `docs/ARCHITECTURE.md` and add a
  one-line pointer to it from `AGENTS.md` Conventions. The existing suites are the
  oracle; new tests only pin the *new* single-source/privacy structure.

## Fail-safe & Boundary Checklist
- **Malformed / empty / null input:** No change. Packs are compile-time interfaces;
  `validateTree`/renderer see the identical resolved `BoxNode` shape and run the
  same code, so malformed-node handling (skip/degrade) is unchanged.
- **Deeply nested / cyclic tree:** No change. No traversal/render code is touched.
- **Offline agent / rapid events / racing agent turn:** No change. No runtime path,
  event loop, or patch apply is modified.
- **Type-drift during extraction (the one real risk):** A dropped/renamed/retyped
  `box` field changes `keyof BoxNode` and trips the type-level field-set
  preservation guard under `pnpm typecheck`; a field left as a direct member
  instead of moving into its pack trips the new source-AST single-source assertion
  under `pnpm test`. Both fail the build before merge (DC-003).
