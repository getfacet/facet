## Overview

- Feature slug: `ag-ui-event-layer`
- Intake brief: `specs/feature-intake/ag-ui-event-layer.md`
- Context evidence: `specs/context/ag-ui-event-layer.md`
- Affected `@facet/*` packages: `@facet/ag-ui` (new), `@facet/runtime`
  (`RuntimeFrameContext` for frame-stage snapshots), `@facet/server` validation
  hardening, docs/package metadata
- One-paragraph technical approach: add a new adapter package at
  `packages/extensions/ag-ui` that uses official `@ag-ui/core` and
  `@ag-ui/client` as the public AG-UI edge, while keeping Facet's canonical stage
  and runtime contract unchanged. Facet stage updates are encoded under the
  reserved AG-UI state path `/facet/stage` as `STATE_DELTA` or
  `STATE_SNAPSHOT`; inbound AG-UI state/custom/raw payloads are narrowed before
  becoming native `ServerMessage` or `ClientEvent`. Browser consumers can pass an
  `AgUiTransport` to `useFacet`, and server integrators can mount the Node-only
  `@facet/ag-ui/server` handler around a `FacetRuntime`. Review hardening added
  bounded aggregate state-payload guards, redacted browser error logging,
  per-run HTTP agent creation after timeout aborts, and authorization queuing
  that serializes visitor resolution without serializing unrelated resolved
  visitor runtime turns.

## Invariant Fit Audit (Facet - REQUIRED)

| # | Invariant | Touched? | Safe design |
|---|---|---|---|
| 1 | Responsibility boundary (no backend/domain in Facet) | N/A | AG-UI tool/activity/reasoning events may be transported or ignored, but the adapter does not execute domain tools or add backend fetch policy. |
| 2 | Mechanism vs policy | Yes | `@facet/ag-ui` standardizes the event envelope only. Agent behavior, page brief, tool choice, and stage authoring remain owned by the consuming agent/runtime. |
| 3 | Fail-safe (never throw/inject; degrade to plain) | Yes | Only `/facet/stage` state deltas/snapshots are converted to Facet patches. Unknown AG-UI events are ignored. Malformed snapshots/deltas return no messages or no-op patches; renderer still receives native `ServerMessage` and folds through `foldPatchIntoStage`. |
| 4 | Declarative + tokens only | N/A | No brick, token, raw HTML, raw CSS, JS, or pixel escape hatch is added. AG-UI state contains Facet `FacetTree` only under the reserved namespace. |
| 5 | Flow-only safety (overlay = dedicated brick) | N/A | No visual/layout capability is introduced. |
| 6 | Two-writers coherence (ordering/version rule) | Yes | `AgUiTransport` owns one monotonically increasing local `seq` and one promise chain. Every `send` and `record` is converted into `RunAgentInput.forwardedProps.facet.{visitor,event|record}` after that seq is stamped, so send-order is the only ordering source. `runFacetAsAgUi` copies that `seq` into the native `ClientEvent` / `CollectedEvent`, ignores `RunAgentInput.state` for stage authority, and delegates serialization to `FacetRuntime` for the `(agent, visitor)` lane. Server output for one run is an ordered AG-UI event stream: `RUN_STARTED`, then mapped Facet batches in runtime delivery order, then `RUN_FINISHED`/`RUN_ERROR`. Duplicate snapshots are root replaces and therefore coherent; the adapter does not claim delta replay/resume, so reconnect repair is by fresh `STATE_SNAPSHOT`, not by re-applying old deltas. Local `record` uses `forwardedProps.facet.record` and never invokes the agent. |
| 7 | Backend via agent tools, never client fetch | N/A | Client fetches only the AG-UI server endpoint. Business API/tool execution stays in the agent process outside Facet. |

## Fail-safe & Boundary Checklist

- Malformed / empty / null input: `RunAgentInputSchema.safeParse` guards HTTP input; invalid `forwardedProps.facet` yields `RUN_ERROR` or a no-op record path, never a runtime call with unknown shape. Tests: WU-1 `events.test.ts` covers null/non-object/non-Facet events; WU-3 `server.test.ts` covers malformed HTTP input.
- Deeply nested / cyclic tree: snapshots/deltas are JSON payloads from AG-UI, but direct helper calls still treat unknown/cyclic values as untrusted. Stage writes go through `foldPatchIntoStage` / `validateTree`, which returns a valid tree and never throws to the renderer. Tests: WU-1 covers hostile snapshot/delta and direct cyclic object input; WU-2 covers client fold of invalid stage payload.
- Offline agent / rapid events / racing agent turn: the adapter delegates turn ordering to `FacetRuntime`; browser `AgUiTransport` serializes submitted runs to preserve current send-order semantics. Tests: WU-2 covers rapid send/record serialization; WU-3 covers agent throw/offline-style runtime failure returning `RUN_ERROR` and ordered rapid requests.
- Duplicate/replay: the AG-UI adapter does not implement native SSE resume. Duplicate `STATE_SNAPSHOT` is coherent because it root-replaces `/facet/stage`; duplicate `STATE_DELTA` is not promised as replay-safe and is repaired by a fresh snapshot. Tests: WU-2 covers duplicate snapshots not duplicating chat and not crashing; WU-3 covers fresh snapshot output.
