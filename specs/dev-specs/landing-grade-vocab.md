## Overview
- Feature slug: `landing-grade-vocab`
- Intake brief: `specs/feature-intake/landing-grade-vocab.md`
- Context evidence: `specs/context/landing-grade-vocab.md`
- Affected `@facet/*` packages: `@facet/core`, `@facet/react`, `@facet/assets`
- One-paragraph technical approach:
  Widen Facet's CLOSED token vocabulary so an agent can compose an
  Apple/Browserbase-grade landing page without any raw CSS, pixels, hex, or
  absolute positioning. We (a) extend `FONT_SIZES` with `4xl/5xl/6xl` and add
  eight NEW closed token groups тАФ `MIN_HEIGHTS`, `MAX_WIDTHS`, `TRACKINGS`,
  `LEADINGS`, `GRADIENTS`, `SCRIMS`, `SCHEMES`, `HIGHLIGHTS` тАФ plus a boolean
  `sticky`; (b) grow `BoxStyle`
  (`minHeight`/`maxWidth`/`sticky`/`gradient`/`backdropScrim`/`scheme`),
  `TextStyle` (`tracking`/`leading`/`highlight`), and `BoxNode` with a
  `backdrop?: NodeId` reference; (c) validate every new style token by array
  membership in `primitive-node-validation.ts` (drop-with-issue on unknown) and
  carry `backdrop` as a string node-id resolved fail-safe at render time; (d)
  make every new group operator-overridable through the existing theme
  machinery; (e) paint the `backdrop` as exactly TWO renderer-owned layers тАФ the
  referenced MEDIA node re-rendered through the existing `isSafeMediaSrc`-gated
  `renderMediaNode` as an absolutely-positioned cover layer, with flow children
  on top in normal flow тАФ and resolve `scheme` as a bounded subtree color-map
  swap. No new brick, no raw CSS in the tree, no author-settable
  position/z-index/offset.

## Concrete token design (pinned)

CLOSED token members added (all concrete values live in a theme, never in the tree):

- `FONT_SIZES` (EXTEND): add `4xl`, `5xl`, `6xl` тЖТ `xs sm md lg xl 2xl 3xl 4xl 5xl 6xl`.
- `MIN_HEIGHTS = ["auto","half","screen"]` тЖТ `BoxStyle.minHeight` (theme: `auto`, `50svh`, `100svh`).
- `MAX_WIDTHS = ["none","prose","narrow","wide"]` тЖТ `BoxStyle.maxWidth` (theme: `none`, `65ch`, `640px`, `1200px`).
- `TRACKINGS = ["tight","normal","wide"]` тЖТ `TextStyle.tracking` (letter-spacing).
- `LEADINGS = ["tight","normal","relaxed"]` тЖТ `TextStyle.leading` (line-height).
- `GRADIENTS = ["none","accent","dusk","dawn"]` тЖТ `BoxStyle.gradient` (theme: CSS `linear-gradient(...)`, `none`).
- `SCRIMS = ["none","light","dark"]` тЖТ `BoxStyle.backdropScrim` (theme: overlay bg over the backdrop layer).
- `SCHEMES = ["light","dark"]` тЖТ `BoxStyle.scheme` (subtree color-map swap).
- `HIGHLIGHTS = ["none","accent","band"]` тЖТ `TextStyle.highlight` (theme: decoration/background behind the text run).
- `sticky?: boolean` тЖТ `BoxStyle.sticky` (renderer `position:sticky` + framework-owned top offset; no author offset).
- `backdrop?: NodeId` тЖТ `BoxNode` (a node-id STRING; resolved to a MEDIA node fail-safe at render time тАФ NOT a token).

New theme-overridable groups on `FacetTheme` + `ResolvedTheme`: `minHeight`,
`maxWidth`, `tracking`, `leading`, `gradient`, `scrim`, `highlight`, and
`colorDark` (the dark-scheme palette). `sticky` is boolean with a
framework-owned offset (no theme group). `scheme` selects between
`ResolvedTheme.colorLight` (= the base `color` map) and
`ResolvedTheme.colorDark` (no separate group beyond `colorDark`).

### `scheme` mechanism (highest-surface item тАФ pinned)

`ResolvedTheme` gains `colorDark: Record<Color,string>` and `colorLight:
Record<Color,string>` (defaults to the base `color` map); `color` stays the
ACTIVE map (defaults to `colorLight`). In `renderer-render.tsx` the box case,
when `style.scheme` is set, children are rendered with a derived theme
`{ ...theme, color: scheme === "dark" ? theme.colorDark : theme.colorLight }`.
This is a pure, bounded per-subtree swap (renderer-owned, read-only): it flips
ONLY the color-token map for that box's subtree, cannot leak upward, and a
nested `scheme:"light"` island restores `colorLight`. Unknown scheme тЖТ the map
is unchanged (light default).

### `backdrop` node-consumption semantics (pinned тАФ RISK-INV-4)

`backdrop` references a STANDALONE media node id; it is NOT required to be a
flow child. The renderer paints the resolved media as the background layer
regardless of whether the same id also appears in `children`; it performs NO
de-duplication тАФ an id present in both `backdrop` and `children` renders in BOTH
places (once as the bg layer, once in flow). This is deterministic and additive,
keeping the agent-authored tree and renderer resolution coherent (invariant #6).

### `RecipeBoxStyle`/`RecipeTextStyle` (waived for v1 тАФ RISK-API-3)

The new style fields are NOT mirrored into `RecipeBoxStyle`/`RecipeTextStyle` or
`theme-recipe-validation.ts` in v1. Landing-grade tokens are node-style-only
(not settable via component-variant recipes). This is an additive capability
gap, not a consumer break; deferred as a follow-up if recipe authors need it.

## Fail-safe & Boundary Checklist
- Malformed / empty / null input: unknown style token value тЖТ dropped WITH an
  issue, node keeps rendering (`asToken`/`asBool`); `backdrop` non-string тЖТ
  dropped; empty/whitespace `src` on the resolved media тЖТ `isSafeMediaSrc`
  fails тЖТ no layer painted; `validateTree` and the renderer never throw.
- Deeply nested / cyclic tree: `backdrop` resolves to a MEDIA node only (media
  has no children) so no recursion is possible; a `backdrop`тЖТbox or
  `backdrop`тЖТself reference paints NO layer; the resolved backdrop node
  decrements `budget.left` so it counts against `MAX_RENDER_NODES`; `MAX_DEPTH`
  unchanged.
- Offline agent / rapid events / racing agent turn: **N/A-with-justification (no
  bespoke test)** тАФ this feature adds no new writer, event, interaction, or local
  execution surface; `scheme` and `backdrop` are pure read-only render-time
  resolutions and the server stays the sole stage writer, so tokens/attributes
  ride existing patches through the unchanged pure `applyPatch`. The existing
  patch-ordering/coherence tests already cover that path; a bespoke test here
  would only assert the absence of a code path.
