## Overview
- Feature slug: `tree-data-bindings`
- Intake brief: `specs/feature-intake/tree-data-bindings.md`
- Context evidence: `specs/context/tree-data-bindings.md`
- Affected `@facet/*` packages: `@facet/core`, `@facet/react`, `@facet/agent-tools`,
  `@facet/reference-agent`, `@facet/agent`, and — regression-only (no production
  edit) — `@facet/runtime`, `@facet/server`, `@facet/client`, `@facet/ag-ui`.
- One-paragraph technical approach: Add an optional per-tree `data` warehouse
  (`Record<datasetName, Dataset>` where `Dataset = Array<Record<string,
  string|number|boolean>>`) and an optional per-node `from` binding on the six
  data-bearing node types (`table`, `chart`, `list`, `keyValue`, `metric`,
  `stat`). A single pure `@facet/core` module (`data-binding.ts`) owns BOTH the
  hostile-input sanitizer (`sanitizeDataWarehouse`) and the ONE shared
  resolve-precedence-and-project helper (`resolveNodeData`) — the latter is
  called from BOTH the content gate (`tree.ts`) and every renderer
  (`@facet/react`) so "shows something" and the actual render can never diverge
  (RISK-INV-5). `data` is sanitized inside `validateTreeUnsafe` and copied onto
  the returned tree so the ONE fold (`foldPatchIntoStage`) that runs identically
  on server and client keeps it (RISK-INV-1). Inline arrays stay REQUIRED and
  `from`/`data` are purely additive optionals (RISK-API-2 → option **a**), so
  every existing unconditional accessor is untouched. `from` is a bounded dataset
  NAME only — never a URL/source/resolver (RISK-INV-2). Resolution is a pure
  render-time lookup with NO client-side data writer (RISK-INV-4). Dangling /
  absent / malformed `from` renders an EMPTY node and counts as non-content, and
  a later `/data/<name>` patch fills it in (buffered forward reference).

### Projection Contract (the closed, renderer-owned mapping `resolveNodeData` implements)

`resolveNodeData(node, warehouse)` returns the effective, inline-shaped value for
a node. Precedence is static: **`from` present ⇒ project the named warehouse
dataset (ignore inline); `from` absent ⇒ return the node's inline value.** A
`from` that names an absent/empty/malformed dataset yields the node type's EMPTY
value (`[]` / no scalar). Projection is fixed and non-configurable beyond the
minimal selectors below (no DSL, no computed columns — see Invariant #2):

- **table**: rows = the dataset rows as-is (row-records already match the table
  row shape); the node's own `columns[].key` select which cells show. No new field.
- **chart**: one series per NUMERIC dataset column (series `label` = column key,
  `values` = that column across rows), capped at `MAX_CHART_SERIES` /
  `MAX_CHART_POINTS`. Non-numeric columns are ignored. No new field.
- **list**: each row → one item; `title` = first string-valued column, `body` =
  second string-valued column (both by dataset column order). No new field.
- **keyValue**: each row → one `{label, value}`; `label` = first column, `value` =
  second column (by order); rows lacking both are dropped. No new field.
- **metric / stat** (the scalar case the Decision Lock routed here): reads ONE
  cell via a small closed selector — new optional `column?: string` (dataset
  column supplying `value`) and `row?: number` (row index, default `0`, clamped
  to `[0, MAX_TABLE_ROWS)`). `label` stays inline (a caption). Missing
  `column`/cell ⇒ empty node. `column`/`row` are the ONLY new node fields beyond
  `from`; they are bounded (`column` a slot-name string, `row` a finite
  non-negative integer), not an expression.

## Fail-safe & Boundary Checklist
- **Malformed / empty / null input**: `data` not an object ⇒ dropped whole with an
  issue. A dataset value that is not an array, or a row that is not a plain
  object, or a cell that is not string/number/boolean ⇒ that dataset/row/cell
  dropped, survivors kept. `from` not a conforming name string ⇒ dropped with an
  issue, node stays and falls back to inline/empty. Never throws.
- **Deeply nested / cyclic / oversized data**: cells are scalar-only, so nested
  objects/arrays are dropped (non-recursive — no cyclic data possible). Caps:
  `MAX_DATASETS` datasets, `MAX_DATASET_NAME_CHARS` name length, `MAX_TABLE_ROWS`
  rows/dataset, `MAX_TABLE_COLUMNS` columns/row, `MAX_TABLE_CELL_CHARS` cell
  chars. Forbidden keys (`__proto__`/`constructor`/`prototype`, fetch-like) and
  non-slot-name column keys dropped. A 10^6-row array is clamped to `MAX_TABLE_ROWS`.
- **Offline agent / rapid events / racing agent turn**: `data` rides existing
  patches through the shared `applyPatch`/`foldPatchIntoStage` — same ordering,
  salvage, and no-drift guarantees as `nodes`. A `/data/<name>` patch that lands
  before the node, or a node bound before its data lands, both converge (buffered
  forward reference, DC-008). Seed gate (`isSeedableTree`) and offline gate
  (`hasBuiltStage`) inherit the corrected content predicate: a populated
  data-bound initial/built stage is now correctly seedable/not-overwritten.
