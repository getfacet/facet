## Overview

- Affected packages: documentation for all published `@facet/*` packages; no
  package runtime or public API changes
- Technical approach: turn the root README into a short decision-and-routing
  page, give Getting Started, Design System, and Agent Integration their own
  canonical guides, narrow existing reference documents back to one job each,

  specs stay unchanged behind one archive authority notice.

## Fail-safe & Boundary Checklist

- Malformed / empty / null input: no runtime behavior changes. Documentation
  explains that author mutations reject atomically, stale render input degrades
  fail-soft, and a malformed custom Theme falls back whole. Existing regression
  coverage: `packages/agents/agent-tools/src/executor.test.ts`,
  `packages/core/core/src/validate.test.ts`, and
  `packages/core/runtime/src/assets.test.ts`.
- Deeply nested / cyclic tree: no validator change. Architecture remains the
  canonical source for traversal caps, cycle breaking, and valid-sibling
  survival; entry docs link rather than restate implementation detail. Existing
  regression coverage: `packages/core/core/src/validate.test.ts`.
- Offline agent / rapid events / racing agent turn: no event-loop change.
  Getting Started must not promise availability beyond the reference transport;
  Architecture remains authoritative for per-visitor serialization. Existing
  regression coverage: `packages/adapters/agent-client/src/connect.test.ts` and
  `packages/core/runtime/src/runtime.test.ts`.
- Documentation input: broken paths, missing anchors, invalid canonical GitHub
  document URLs, or type errors in opt-in concrete snippets fail the docs gate
  with source file and target evidence. Ordinary illustrative fragments must be
  labeled pseudocode and are not silently treated as executable examples.
