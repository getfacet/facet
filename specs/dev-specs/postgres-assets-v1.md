## Overview
- Feature slug: `postgres-assets-v1`
- Intake brief: `specs/feature-intake/postgres-assets-v1.md`
- Context evidence: `specs/context/postgres-assets-v1.md`
- Affected `@facet/*` packages: `@facet/store-postgres` (all production change), `@facet/runtime` (consumed only — `AssetsStore`/`AssetDocuments`/`loadAssets` reused, interface unchanged)
- One-paragraph technical approach:
  Add a third Postgres adapter, `PostgresAssets`, to `@facet/store-postgres` so the
  operator's per-agent custom themes/stamps/initial-tree become durable — closing the
  one asymmetry among Facet's three persistence seams (`StageStore` and `Sink` already
  have Postgres backends; `AssetsStore` did not). `PostgresAssets` mirrors
  `PostgresStageStore`/`PostgresSink` exactly: a `constructor(private readonly pool: Pool)`,
  a read `load(agentId)` that returns **RAW** `AssetDocuments` from a single
  `facet_assets` row (no `@facet/core` validation — `loadAssets` stays the single gate),
  and a concrete **write** `putAssets(agentId, docs)` that upserts per agent. The shared
  `initSchema(pool)` is extended with an idempotent `create table if not exists
  facet_assets`, and the class is re-exported through the package barrel. No change to
  the `AssetsStore` interface, no new dependency (`pg` stays a peer dep).

## Fail-safe & Boundary Checklist
- **No row for agent:** `load` returns `{ themes: [], stamps: [] }` (no `initialTree`); `loadAssets` still resolves the default base layer (DC-002).
- **Null jsonb columns (`themes`/`stamps` NULL):** normalized to `[]` in `load` (RISK-INV-2); never `null` fields that could throw at `loadAssets`' spread sites.
- **Missing `initial_tree` (NULL):** `initialTree` is left unset (only set when the column is non-null), mirroring `FileAssets` (`file-assets.ts:48-50`). No empty-tree seed → EMPTY_TREE trap never armed by the store.
- **Malformed stored jsonb shape (e.g. a stamp object missing `name`):** returned RAW; dropped by `loadAssets`' per-doc validator with a recorded issue; other docs survive; no throw (DC-004). `pg` parses jsonb, so no store-side parse throw is possible.
- **DB down / `pool.query` rejects at boot:** rejection propagates out of `load`, is caught by `loadAssets`' try/catch → `{ themes: [], stamps: [], issues: ["assets load failed: …"] }` → defaults resolve; the quickstart caller logs issues and continues boot (RISK-INV-4, resolution (a)).
- **`initSchema` run repeatedly:** idempotent `create table if not exists facet_assets` — safe (DC-005).
- **Deeply nested / cyclic tree:** not the store's concern — `initial_tree` is opaque jsonb on the wire; `validateTree` in `loadAssets` bounds/normalizes it before any seed.
- **Concurrent writes to the same agent:** `putAssets` is an upsert `on conflict (agent_id) do update` — last-writer-wins per agent row, consistent with `PostgresStageStore.save`.
