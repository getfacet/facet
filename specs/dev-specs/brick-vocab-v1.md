## Overview

- Feature slug: `brick-vocab-v1`
- Intake brief: `specs/feature-intake/brick-vocab-v1.md`
- Context evidence: `specs/context/brick-vocab-v1.md`
- Affected `@facet/*` packages: `@facet/core`, `@facet/react`, `@facet/quickstart`
  (+ `apps/playground` demo/consumer)
- One-paragraph technical approach: Grow the drawing vocabulary **without growing
  the brick count** (stays 4) by mirroring how `field` already discriminates on
  `input`. **`image` becomes `media`** — one node typed by
  `kind:"image"|"video"` (`MEDIA_KINDS` single-source const in `nodes.ts`),
  `ImageNode`/`ImageStyle`→`MediaNode`/`MediaStyle`, with video-only optional
  props (`poster?`, `controls?`); `isSafeImageSrc`→`isSafeMediaSrc` (same
  never-`javascript:`/never-`data:text/html` allowlist) gates `media.src` **and**
  `media.poster` at both the validate and render seams; a legacy `{type:"image"}`
  blob **normalizes** to `{type:"media",kind:"image"}` in `validateTree` and the
  raw renderer keeps a thin `case "image"` alias so old persisted/patched trees
  still render (DC-010). **`FIELD_INPUTS`** grows by
  `checkbox|radio|select|switch` (the type auto-derives; the validator's
  `asToken` picks them up free) plus a new `field.options?: readonly string[]`
  (sanitized string[], empty-safe, count/length-capped); the react `case "field"`
  arm **branches on input** to emit **native** controls (`<select>`+`<option>`,
  `<input type="checkbox|radio">`, a `role="switch"` checkbox) — the browser owns
  the select popup, so **no overlay/z-index brick** is introduced and flow-only
  holds (invariant #5). The collect READER is broadened from
  `input[data-facet-field-id]` to `[data-facet-field-id]` and reads **per control
  kind** — `<select>.value`, checkbox/switch `.checked`, and the **checked**
  member of a radio group (not first-mounted) — preserving the collect-at-press,
  uncontrolled-DOM model (invariant #6). Two style tokens land on `BoxStyle`:
  **`scroll`** widens `boolean`→`ScrollAxis|true` (`SCROLL_AXES=["x","y"]`, legacy
  `true`=`"y"`) with the theme branching `x`→a **bounded** horizontal carousel and
  `y`/`true`→today's vertical region; and a new **`columns`** numeric token group
  (`COLUMNS=[2,3,4]`) switches the box from `display:flex` to `display:grid`. All
  new surfaces enter through **both** validation layers (stored `validateTree` /
  `foldPatchIntoStage` / `validateStamp` AND the raw live-patch renderer path),
  strip unknown values to plain flow **with an issue** (never throw, never
  inject), and are taught by `STAGE_SPEC`, pinned by a cross-package drift net.

### Decision 1 — `image`→`media` shape and legacy normalization (RISK-API-1, RISK-INV-6, DC-001/002/010)

`MediaNode` = `{ id, type:"media", kind:"image"|"video", src, alt?, poster?,
controls?, style?:MediaStyle }`. `MEDIA_KINDS = ["image","video"] as const` lives
in `nodes.ts` beside `FIELD_INPUTS` (both are node discriminators — single source
for validator + renderer). `MediaStyle` = the former `ImageStyle`
(`radius`/`width`/`ratio`). Video-only props: `poster?: string` (a still URL,
gated by `isSafeMediaSrc`), `controls?: boolean`. **Deferred** (locked out of v1,
per brief Constraints): `audio` kind, `autoplay`/`loop`/`muted` (dodges the
autoplay-policy question). Validation (`sanitizeNode` `case "media"`):
`src = asString(raw.src)`; missing → drop node with issue (DC-002); unsafe scheme
→ drop with issue; `kind = asToken(raw.kind, MEDIA_KINDS)` — **absent kind
defaults to `"image"`** (friendly + legacy), a **present-but-unknown** kind (e.g.
`"gif3d"`) drops the node with an issue (DC-002); `alt` optional (default `""` —
video needs no alt text); `poster` kept only if a string **and** `isSafeMediaSrc`
(else omitted); `controls` kept only if a literal boolean. **Legacy normalize
(DC-010):** a `{type:"image",src,alt}` blob is normalized in `sanitizeNode` to the
`media`/`kind:"image"` shape (a `case "image"` alias arm before/within the media
arm), so old server-persisted trees survive `validateTree`. The raw renderer
(`StageRenderer.tsx`, bypasses validateTree) keeps a thin `case "image"` alias
that renders identically to `media`/kind:image, so a legacy live-patch node also
renders. `FacetNode` union swaps `ImageNode`→`MediaNode`; the `export *` barrels
flow the rename automatically (no barrel edit).

### Decision 2 — expanded field inputs + `options` + native-control render (RISK-API-3, RISK-INV-2, DC-003/005)

`FIELD_INPUTS = ["text","number","email","password","search","checkbox","radio","select","switch"] as const`
(type auto-derives; `validate.ts:351` `asToken<FieldInput>(raw.input, FIELD_INPUTS)`
picks the new members up free). `FieldNode` gains
`options?: readonly string[]`. Validation adds an `options` sanitizer in the
`field` arm: keep only string members, drop non-strings, cap count at
`MAX_FIELD_OPTIONS` (new local const, 64) and each item at `MAX_FIELD_VALUE_CHARS`
(reuse the existing cap); an empty/absent array is safe (omitted). Renderer
`case "field"` **branches on `input`** (the single-`<input>` assumption at
`StageRenderer.tsx:1001` is the defect — `type="select"|"switch"` are invalid
HTML input types that silently degrade to text):
`select` → `<select data-facet-field-id={id}>` with an `<option>` per
`options`; `radio` → **N** `<input type="radio" name={node.name}
value={opt} data-facet-field-id={id}>` (one per option, sharing the node id);
`checkbox` → `<input type="checkbox" data-facet-field-id={id}>`; `switch` →
the SAME native `<input type="checkbox" role="switch" data-facet-field-id={id}>`
styled as a toggle (brief's stated assumption — a styled boolean, **no** custom
element, **no** overlay); the text-family inputs render as today; unknown/absent
`input` → `text` (existing fallback at `:983-985`, DC-005). **Invariant #5 gate:**
the ONLY popup (the select dropdown) is rendered by the browser outside the flow;
no node, style, or CSS introduces `position:absolute`/`z-index` for any popup.

### Decision 3 — collect reader extended for the new controls (RISK-INV-3, RISK-INV-4, RISK-API-4, DC-004)

`collectFieldValues` (`StageRenderer.tsx:132-218`) keeps its uncontrolled-DOM,
collect-at-press model (invariant #6) — only the DOM READER changes. (a) The
selector broadens from `root.querySelectorAll("input[data-facet-field-id]")` to
`root.querySelectorAll("[data-facet-field-id]")` so `<select>` elements match
(they are not `<input>`). (b) The `inputByNodeId` single-element map becomes an
`elementsByNodeId: Map<string, Element[]>` — a radio group stamps N elements onto
ONE node id, and the reader must see all of them. (c) The value read at
`:202-216` special-cases by element/control kind: `<select>` → `.value`;
`checkbox`/`switch` → `.checked` ⇒ `"true"` (or omit the key when unchecked —
locked below); a **radio group** (multiple elements for the same node id) →
the value of the **checked** member (skip unchecked; none checked ⇒ omit),
NEVER first-mounted-wins (RISK-INV-4); text-family `<input>` → `.value` as
today. The password exclusion (`:164-166`), the `MAX_FIELD_VALUE_CHARS`
per-value cap, the `MAX_FIELDS_KEYS` count cap, and the mounted-shadowing rule
(a hidden earlier same-named field must not shadow a visible later one) all stay
intact. **Locked:** unchecked checkbox/switch and an all-unchecked radio group
**omit** the key (rather than emit `""`) — a boolean control's "off" is absence,
matching how a blank text field already omits nothing but a real value; the agent
reads presence. Documented in the collect DC test.

### Decision 4 — `scroll` axis, bounded horizontal region (RISK-INV-1, RISK-API-2, DC-007)

`SCROLL_AXES = ["x","y"] as const` in `tokens.ts`; `type ScrollAxis`.
`BoxStyle.scroll?: ScrollAxis | true` (legacy `true` = vertical, back-compat).
Validation replaces the `asBool` branch (`validate.ts:234-239`, which today
STRIPS `"x"`/`"y"`) with an axis-normalizer: accept `"x"`/`"y"` (kept), map legacy
`true`→`"y"` (kept), strip anything else **with an issue** (DC-008). Theme
`boxStyle` (`theme.ts:164-169`) branches: `scroll === "y" || scroll === true` →
today's exact CSS (`overflowY:"auto"`, `overflowX:"hidden"`,
`maxHeight:SCROLL_MAX_HEIGHT`, `minHeight:0`); `scroll === "x"` →
`overflowX:"auto"`, `overflowY:"hidden"`, `maxWidth:"100%"`, `minWidth:0`.
**RISK-INV-1 (consciously superseding the `// Never overflow-x (RISK-INV-6a)`
comment at `theme.ts:166`):** the `x` case is a **BOUNDED internal scroll
region**, not a relaxation of flow-only — `maxWidth:100%` + `minWidth:0` keep the
box within its parent so children clip **inside** the box and no ancestor gains a
horizontal scrollbar (the page never widens). The box still cannot overlap
siblings. The spec cites the RISK-INV-6a annotation so review sees the earlier
"never overflow-x" decision is being **superseded for the deliberate `x` axis**,
not accidentally broken; the comment is rewritten to say so.

### Decision 5 — `columns` grid token (RISK-API-5, RISK-INV-5, DC-006)

`COLUMNS = [2, 3, 4] as const` (numeric token group) in `tokens.ts`;
`type Columns = 2 | 3 | 4`. `BoxStyle.columns?: Columns`. Validation coerces a
number or numeric-string, membership-checks against `COLUMNS`, else strips **with
an issue** (DC-008) — a numeric-token variant of the existing `asToken` gate (the
brief example emits `columns:3` as a number). Theme `boxStyle` (`theme.ts:145`,
which hardcodes `display:"flex"`): when `columns` is present the box switches to
`display:"grid"` + `gridTemplateColumns: repeat(N, minmax(0,1fr))`; `gap`/`pad`/
`bg`/`radius`/`border`/`width` still apply; **`direction`/`wrap` are ignored in
grid mode** (grid owns the axis — documented). Grid is flow layout (no absolute
positioning), so invariants #4/#5 hold. `columns` may combine with `scroll` (a
scrollable grid). `Sizing` (width `auto|full`) is intentionally **not** widened —
grid is the new N-column mechanism, so width stays additive-free.

### Decision 6 — `isSafeImageSrc`→`isSafeMediaSrc` (RISK-INV-6)

Rename the URL-scheme gate to `isSafeMediaSrc` (same allowlist: `https://`,
`http://`, `//`, `data:image/`, root-relative `/…`). Applied to `media.src` AND
`media.poster` at **both** the validate seam (`sanitizeNode` media arm) and the
render seam (`StageRenderer.tsx` media arm). It keeps the SAME
never-`javascript:`/never-`data:text/html` boundary so a video `src` stays a
static URL exactly like an image (invariant #1 — no fetch/data-binding). The gate
still allows `data:image/` for a video element — cosmetically loose but **not a
security hole** (its job is blocking script/html schemes, which it does); widening
to `data:video/` is deferred (the brief keeps `src` a static URL). The barrel
`export *` flows the rename; the one internal consumer (`validate.ts:330`) and the
renderer consumer (`StageRenderer.tsx:973`) update in lockstep within their WUs.

## Fail-safe & Boundary Checklist

- **Malformed / empty / null input:** `media` with no `src` → node dropped (issue),
  never throws. `media` `kind:"gif3d"` → dropped (issue). `media` `src:"javascript:…"`
  → dropped by `isSafeMediaSrc`. `field` `input:"colorwheel"` → validate omits
  `input`, renderer renders `text`. `field` `options:[1,{},"ok"]` → sanitized to
  `["ok"]`; `options:[]`/absent → safe (no `<option>`, `<select>` renders empty).
  `box` `scroll:"sideways"` / `scroll:5` → stripped with issue → normal flow.
  `box` `columns:"lots"` / `columns:9` → stripped with issue → flex flow.
  `validateTree(null | 42)` behavior unchanged. Legacy `{type:"image",src,alt}`
  → normalized to `media`/kind:image (validate) or rendered via the raw `case
  "image"` alias (renderer).
- **Deeply nested / cyclic tree:** unchanged. The media/field render arms add no
  recursion; `collectFieldValues` keeps its `MAX_DEPTH`/`RENDER_BUDGET`/ancestor-set
  guards — the broadened `[data-facet-field-id]` selector and `Map<string,
  Element[]>` are a single flat DOM enumeration, not a tree walk. A radio group's
  N elements per node id are bounded by `options` count cap.
- **Offline agent / rapid events / racing agent turn:** collect-at-press is
  browser-local; a collect press with the agent offline queues exactly like today
  (per-visitor serial queue unchanged). Rapid presses ⇒ one snapshot per press.
  An agent patch that swaps a field's `input` (e.g. select→text) mid-session
  re-renders the control; the next collect reads whatever is mounted at press time
  (uncontrolled model — no stale value bookkeeping). A racing patch that removes a
  radio's `options` renders zero radios ⇒ that name omits from the next collect.
