## Overview
- Feature slug: `richtext`
- Intake brief: `specs/feature-intake/richtext.md`
- Context evidence: `specs/context/richtext.md`
- Affected `@facet/*` packages: `@facet/core`, `@facet/assets`, `@facet/react`,
  `@facet/agent-tools`, `@facet/reference-agent`
- One-paragraph technical approach: Add a closed, fail-safe leaf primitive
  brick `richtext` whose data is a FLAT `blocks[]` (`paragraph|heading|listItem|
  quote`), each block a list of `runs[]`, each run a string plus a closed
  EXTENSIBLE `Mark[]` tagged union (`bold|italic|underline|strike|code|link`).
  `@facet/core` owns the types, a `validateRichText` sanitizer (clamps
  `level` 1â€“3 and list `depth`, caps `blocks`/`runs`/`marks`/text length, drops
  unknown mark kinds, degrades unknown block types to `paragraph`, routes an
  internal `link` target through the shared `normalizeFacetAction`, and gates an
  external `link` href through a NEW, STRICTER `isSafeHref`), the compiler-
  exhaustive `BRICK_REGISTRY` entry, the barrel re-export of `isSafeHref`, and
  the default-catalog advertisement. `@facet/react` renders it with a bespoke
  `renderRichText` (flow blocks/runs, theme-owned mark look + renderer-owned flow
  indent, internal link dispatched via the single `classifyPress â†’ onPress` writer,
  external link a plain re-gated `<a href rel="noopener noreferrer">`), joins the
  motion snapshot, and keeps the registry-sync guard green. `@facet/agent-tools`
  and `@facet/reference-agent` teach the closed shape and add a stage summarizer.
  No HTML/markdown is ever parsed; only patches travel; the renderer never throws.

## Resolved Assumptions (from the intake Decision Lock)
- **Block-level style type `S`** â€” RESOLVED: `RichTextNode` reuses the existing
  `TextStyle` as its block-level `Styleable<TextStyle>` slot. Block-level
  typography is the same token family as `text`; per-run emphasis is owned by
  `Mark[]`. A dedicated `RichTextStyle` would duplicate `TextStyle` with no v1
  gain; reuse is the minimal additive choice and a future `RichTextStyle` can
  replace `S` additively if block-only tokens ever diverge. No `ActiveLook`
  (owner decision: no active-look on richtext v1) and no `DataBound` (leaf /
  not-store-bound, DC-005).
- **`link` mark shape + `isSafeHref` allowlist** â€” RESOLVED: single mark
  `{ kind: "link"; target: LinkTarget }` where
  `LinkTarget = FacetAction | { readonly href: string }`. Validation branches on
  shape: an object carrying a string `href` is treated as an external URL and
  gated by `isSafeHref`; otherwise the value is routed through the shared
  `normalizeFacetAction` (internal navigate/agent/toggle). `isSafeHref` allowlist
  = `http://`, `https://`, protocol-relative `//`, and local `/path` ONLY â€” it
  REJECTS `data:` and `javascript:` and every other scheme (see RISK-INV-1; it is
  deliberately STRICTER than `isSafeMediaSrc`).
- **Caps** â€” RESOLVED: `MAX_RICHTEXT_BLOCKS = 64`, `MAX_RUNS_PER_BLOCK = 64`,
  `MAX_MARKS_PER_RUN = 8`, run text reuses `MAX_NODE_BODY_CHARS`, heading
  `level` clamped to the closed set `{1,2,3}`, list `depth` clamped to
  `0..MAX_LIST_DEPTH` with `MAX_LIST_DEPTH = 5`. All are integer-clamped with the
  existing `asNumberToken`/slice idiom; over-cap clamps, never throws.

## Closed Vocabulary (v1)
- `BLOCK_TYPES = ["paragraph","heading","listItem","quote"]`; an unknown block
  `type` degrades to `paragraph` (keeps the text), never dropped.
- `MARK_KINDS = ["bold","italic","underline","strike","code","link"]`; an unknown
  mark `kind` is dropped, the run text is kept. `Mark` is a tagged union whose
  members may carry attrs (link carries `target`), so future valued marks
  (highlight/color) and ordered lists / code block add as new union members /
  block types with NO breaking reshape (DC-006).

## Fail-safe & Boundary Checklist
- **Malformed / empty / null input:** non-array `blocks` â†’ empty richtext (no
  blocks); non-object block â†’ dropped; unknown block `type` â†’ `paragraph`;
  non-array `runs` â†’ block with zero runs â†’ block dropped (nothing to render);
  run missing/non-string `text` â†’ skipped; run whose every mark is unknown â†’
  text kept, marks empty; richtext whose every run is invalid â†’ empty (renders
  nothing, never `null`-throws). `null`/`undefined` node fields never dereferenced.
- **Deeply nested / oversized:** `blocks` capped at 64, `runs` at 64/block,
  `marks` at 8/run, text sliced at `MAX_NODE_BODY_CHARS`, heading `level` clamped
  to 1â€“3, list `depth` clamped to 0â€“5. Over-cap CLAMPS/TRUNCATES, never throws.
  richtext is a LEAF (no child node ids) so it cannot participate in a node cycle
  and adds no depth to the tree's node graph.
- **Offline agent / rapid events / racing agent turn:** richtext is pure content;
  an internal link click routes through the existing single dispatch writer with
  the existing navigate/toggle/agent semantics (no new ordering/version rule). An
  external link is default browser navigation. No new async path, timer, or
  client fetch â€” nothing to race.
- **Unsafe href:** `javascript:`, `data:` (incl. `data:image/svg+xml`), and every
  non-allowlisted scheme are dropped at validate time AND re-gated client-side;
  the link degrades to plain text / inert anchor.
