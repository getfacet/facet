### One-paragraph technical approach

Stamps stop being prompt-copy and become server-side, parameterized building blocks.
The load-bearing primitive is a **pure `@facet/core` helper `expandStamp(stamp, params, at, options?)`**
that takes an already-resolved `FacetStamp`, fills its declared **named slots** from
`params`, **remaps every internal node id to a fresh unique id** (disjoint per call),
runs the result through the existing `validateStamp`-grade sanitization, and returns
`{ root, nodes, slots, ids, issues }` ŌĆö never throwing. `FacetStamp` gains an OPTIONAL
`slots?: Record<string,string>` declaration (slot name ŌåÆ default); slot markers are the
**whole-value token `{{slotName}}`** in a string leaf field (no substring interpolation),
recognized only by the core helpers ŌĆö **never a new `FacetNode` field** (keeps the 4-brick
union untouched, RISK-API-1). Two callers consume the primitive: `@facet/agent`'s new
`Stage.useStamp(stamp, params, at)` (in-process path ŌĆö records ops into `pending`, returns
the id map) and `@facet/quickstart`'s new `use_stamp` tool case, which resolves the name
from the **turn's immutable stamps snapshot** and feeds the expanded subtree **through the
existing `createClosureBuffer`** so it lands as one referentially-closed streaming batch
(RISK-INV-1). The quickstart prompt's `stampsSection` is rewritten from full-JSON copy to a
**names + slots + description advertisement** (BREAKING the prompt-copy path). `DEFAULT_STAMPS`
gains slot declarations + markers. `@facet/agent` keeps its `@facet/core`-only dependency;
no new cross-package edge is introduced (RISK-PKG-1/-4).

### Locked open-question resolutions (writer decisions)

- **Slot marker syntax:** an OPTIONAL `slots?: Record<string,string>` map on `FacetStamp`
  (slot name ŌåÆ default value) plus **whole-value markers**: a string leaf field equal to
  exactly `{{slotName}}` is replaced by `params[slotName]` (else the declared default, else
  empty/dropped). **No substring interpolation** (validate-able, no injection into arbitrary
  strings). Markers fill string leaf fields the sanitizer already governs (`text.value`,
  `media.src`/`alt`, `field.label`/`placeholder`/`value`). Resolves RISK-API-1.
- **`use_stamp` return shape:** `{ root: NodeId, slots: Record<string, NodeId>, ids: Record<oldId, NodeId> }`
  ŌĆö the fresh root id, the slot-nameŌåÆfresh-node-id map (DC-007), and the full oldŌåÆnew id
  map so the agent can address non-slotted nodes for follow-up `set_node`/`append`.
- **`at` shape (v1):** `at = { parent: NodeId }` ŌĆö the stamp root is appended as the **last
  child** of `parent` (mirrors today's `append`, which is end-only). **`index` is a v1
  non-goal** (deferred): honoring an insert index on the streaming path would require
  changing the shared `createClosureBuffer` (`/-` end-append), widening blast radius beyond
  this feature. Both emission paths stay identical (append-at-end).
- **id minting:** `expandStamp` mints fresh ids via an injectable `options.mintId?: () => string`
  (default `globalThis.crypto.randomUUID()`), so `@facet/core` stays node-free (no `node:crypto`
  import, RISK-PKG-3) and tests can assert disjointness deterministically.
- **Param sanitize:** slot values reuse `MAX_FIELD_VALUE_CHARS` (2000). A non-string param
  is dropped with an issue; an over-cap string is truncated; a missing slot falls back to its
  declared default, else empty.
- **DC-009 advertisement location:** lives in `@facet/quickstart` `prompt.ts` (`stampsSection`
  rewrite), NOT in the shared `STAGE_SPEC`. `spec.ts` is intentionally **unchanged** (RISK-API-5:
  a generic `use_stamp` line in `STAGE_SPEC` would advertise a tool the bridge/playground paths
  do not implement).
- **`@facet/runtime`:** no code change. The turn's stamps snapshot is already loaded once by
  `loadAssets` and threaded into the quickstart agent; expansion runs inside the existing
  per-`(agent,visitor)` serial lane. Nothing under `packages/runtime/src` is edited (RISK-INV-4,
  RISK-PKG-1). No WU touches it ŌĆö this is an explicit no-op, not an orphan.

## Fail-safe & Boundary Checklist

- **Malformed / empty / null input:** `expandStamp` given a non-`FacetStamp` object (or a stamp
  whose nodes fail brick-shape) ŌåÆ runs through the shared sanitizer; an unusable stamp yields
  `{ nodes:{}, root:undefined-equivalent no-op, issues:[ŌĆ”] }` ŌåÆ caller emits **no ops** (no-op).
  `use_stamp` with an unknown/blank name (quickstart) or an `undefined` stamp (Stage) ŌåÆ no-op +
  observation/issue. A `params` value that is non-string ŌåÆ dropped with issue; over-cap string ŌåÆ
  truncated at `MAX_FIELD_VALUE_CHARS`. Missing declared slot ŌåÆ declared default, else empty.
- **Deeply nested / cyclic stamp tree:** `expandStamp` reuses `sanitizeNodeMap` /
  `pruneDanglingChildren` / `breakCycles` (depth-capped) before remap, so a cyclic or over-deep
  stamp is bounded ŌĆö never an infinite loop, never a dangling ref into the live tree.
- **Offline agent / rapid events / racing agent turn:** expansion runs in the existing per-turn
  serial lane against a per-turn snapshot, so a concurrent asset write between turns can never
  produce a half-applied or non-deterministic subtree; ids are minted disjoint from
  `session.stage` ids AND ids created earlier this turn AND other same-turn expansions.
- **Unknown parent (`at.parent`):** validated against `knownIds` (quickstart) / degrades to
  no-op + issue (Stage) ŌĆö mirrors `append_node`'s nonexistent-parent reject; never orphans a link.
- **Repeated use on one screen:** N calls ŌåÆ N disjoint id-sets (fresh `mintId` per call), no
  RFC-6902 upsert collision (RISK-INV-2, DC-004).
