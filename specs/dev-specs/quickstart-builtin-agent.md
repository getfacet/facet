- One-paragraph technical approach: Three additive layers, one flow. **(1) Field
  wiring:** `AgentAction` gains `collect?: NodeId` (`nodes.ts:37-43`) and the
  `action` arm of `ClientEvent` gains `fields?: Readonly<Record<string, string>>`

  `data-facet-field-id` on each rendered `<input>`, and on an agent press whose
  action declares `collect` it enumerates field names **data-side** (a
  cycle/depth-guarded walk over `tree.nodes[collect].children`) and reads the
  matching inputs from the DOM within the stage root's subtree ‚ÄĒ a press-time
  view-state snapshot that never enters the tree (invariant #6). `onAction`
  widens backward-compatibly to `(action, fields?)`; the server's `isEventBody`
  guard (`server.ts:127-155`) validates `fields` as a string-record under the cap
  (defense in depth ‚ÄĒ untrusted client). **(2) `@facet/quickstart`:** a NEW
  optional reference-brain package (the `@facet/server`-of-brains): a
  `QuickstartProvider` interface with raw-fetch OpenAI/Anthropic adapters, a
  3-layer prompt (‚Ď† `STAGE_SPEC` + fixed output contract, ‚Ď° the deployer's
  `--guide` markdown, ‚ĎĘ event + current stage + last-20 sink turns), an
  LLM-output salvage parser adapted from `apps/playground/src/generator.ts`
  (copied, never imported ‚ÄĒ nothing may depend on playground), a deterministic
  stub brain, and a `facet-quickstart` bin that serves an HTML shell + prebuilt
  browser bundle from its own wrapper HTTP server while proxying the protocol
  routes to an internal loopback `createFacetServer` ‚ÄĒ `@facet/server`'s route
  table is untouched. **(3) `/live-test`:** a repo skill + vitest E2E files
  implementing the 3-tier gate (stub always; real-LLM smoke when quickstart
  sources changed and a key exists, SKIPPED=FAIL only then; both providers
  pre-merge), reusing the `server.test.ts` harness pattern.

### Decision 1 ‚ÄĒ renderer field collection: container ref + data-side enumeration + scoped DOM read

`StageRenderer` wraps its output in `<div style={{display:"contents"}} ref={stageRootRef}>`
(`display: contents` adds no layout box, so flow layout is unchanged). Each
rendered field input (`StageRenderer.tsx:296`) gains `data-facet-field-id={node.id}`.
A module-internal helper in `StageRenderer.tsx` (same convention as
`classifyPress`/`resolveScreenRoot` ‚ÄĒ helpers stay internal, exercised through
component tests):

```ts
function collectFieldValues(
  tree: FacetTree,
  collectId: NodeId,
  root: ParentNode,
): Readonly<Record<string, string>>;
```

- **Enumerate data-side:** walk from `tree.nodes[collectId]` over `children`
  (boxes only descend; ancestor-set cycle guard + `MAX_DEPTH`, mirroring
  `RenderNode`'s own guards), gathering `(id, name)` for every `type === "field"`
  node with a string `name`, in walk (document) order. Duplicate names: first
  occurrence wins (deterministic).
- **Read DOM-side, scoped:** enumerate
  `root.querySelectorAll('input[data-facet-field-id]')` once and match
  `el.getAttribute("data-facet-field-id") === id` per `(id, name)` ‚ÄĒ NO
  `CSS.escape` (jsdom exposes no `window.CSS`, so an escape-based selector
  would throw in the prescribed test environment; attribute comparison also
  sidesteps escaping arbitrary node ids). The node-id key (not `name`) makes
  the read exact and immune to same-named inputs elsewhere on the page. If the same node is mounted more
  than once (screens may share nodes), the FIRST match in DOM order wins ‚ÄĒ
  documented, deterministic.
- **Only mounted inputs are readable (reviewer P2, specified explicitly):**
  collection reads the rendered DOM, so it captures **visible fields on the
  CURRENT screen only**. A collect target that lives on a non-current screen
  yields `{}`; a `hidden`/toggled-hidden field inside the subtree is simply
  absent from the DOM and is omitted from `fields`. This is the documented
  contract (STAGE_SPEC words it for the LLM: keep a form and its submit button
  together, visible, on the same screen), and both cases are pinned by WU-3
  tests.
- **Coerce + cap:** value = `String(el.value).slice(0, MAX_FIELD_VALUE_CHARS)`
  (cap imported from `@facet/core`).
- **Fail-safe:** unknown `collectId`, non-box target, zero fields, or a missing
  DOM input all degrade to omission / `{}` ‚ÄĒ never a throw (DC-002). If the
  stage-root ref is unexpectedly null at press time, collection bails to `{}`
  (no `document`-wide fallback ‚ÄĒ two renderers on one page must not
  cross-read).

`classifyPress`'s agent arm (`StageRenderer.tsx:102-111`) threads a string-typed
`collect` into `ClassifiedPress`; `handlePress`'s agent case
(`StageRenderer.tsx:170-171`) becomes: `collect` declared ‚Üí
`onAction?.(press.action, stageRootRef.current === null ? {} : collectFieldValues(tree, press.collect, stageRootRef.current))`
(always passes a fields object, possibly `{}`); no `collect` ‚Üí `onAction?.(press.action)`
(fields arg `undefined` ‚ÄĒ today's exact behavior).

### Decision 2 ‚ÄĒ `onAction` widening (backward-compatible) + both construction sites in one WU

```ts
export interface StageRendererProps {
  readonly tree: FacetTree;
  readonly onAction?: (
    action: FacetAction,
    fields?: Readonly<Record<string, string>>,
  ) => void;
}
```

Adding an optional trailing parameter to a callback PROP is source-compatible in
TypeScript (a narrower `(action) => void` handler remains assignable), so every
existing consumer compiles unchanged. The two playground construction sites the
context flags (RISK-API-2: `App.tsx:127`, `live.tsx:62`) are updated **in the
same WU as the renderer change** to thread fields onto the event
(`exactOptionalPropertyTypes`-safe):

```ts
const onAction = (action: FacetAction, fields?: Readonly<Record<string, string>>): void => {
  send(fields === undefined ? { kind: "action", action } : { kind: "action", action, fields });
};
```

### Decision 3 ‚ÄĒ page serving (RISK-PKG-3): wrapper server + loopback proxy; `@facet/server` untouched

`createFacetServer` constructs and owns its own `http.Server`, answers exactly
`/health`, `/stream`, `/event`, `/agent/*`, and 404s everything else
(`server.ts:497-583`); it exports no request handler to compose with. The two
options were (a) run the facet server on an internal loopback port and put a
thin proxy in front, or (b) export the handler / add routes to `@facet/server`.
**Chosen: (a).** (b) would grow the reference transport's public surface for one
consumer's convenience ‚ÄĒ exactly the scope creep the brief forbids. The
loopback hop is same-process localhost (negligible), the proxy is ~40 lines of
`node:http`, and SSE survives because the proxy **pipes** (never buffers):
`req.pipe(proxyReq)` / `proxyRes.pipe(res)`. The proxy forwards **both
directions faithfully**: request method + request headers are copied onto
`proxyReq` (hop-by-hop headers excluded), so `Last-Event-ID` resume
(`server.ts:530`) and `Content-Type` survive the hop; response status + headers
pass through unchanged.

**Internal-port exposure (reviewer P1):** `createFacetServer` has no host
option today and `server.listen(port)` binds ALL interfaces ‚ÄĒ an "internal"
facet server on a random port would expose an unauthenticated `/agent/*`
channel to the network (agent takeover). Three-part fix, all required:

1. **WU-4 adds `host?: string` to `FacetServerOptions`** (additive one-liner:
   `listen(port, host)` when provided; nothing else changes). The quickstart
   boots the internal server with `host: "127.0.0.1"` so it is loopback-only.
2. **Random per-boot `agentToken`** (`crypto.randomUUID()`) is passed to the
   internal `createFacetServer` ‚ÄĒ defense in depth even on loopback
   (`server.ts:550` enforces it when set).
3. **The wrapper does NOT proxy `/agent/*`** ‚ÄĒ those paths answer 404 at the
   wrapper. Quickstart's brain is in-process; external dial-in is an
   "advanced" jack and out of quickstart's surface. (Tier-1 E2E pins this.)

Concretely, in `packages/quickstart/src/server.ts`:

- `startQuickstart(options)` boots `createFacetServer({..., host: "127.0.0.1",
  agentToken: <random>})` at a random high port with a bind-retry loop (the
  `server.test.ts:21-35` pattern), then a public wrapper `http.Server` on
  `options.port` (default 5292):
  - `GET /` ‚Üí HTML shell (template string in TS: `<div id="root">` +
    `<script type="module" src="/app.js">`; no external assets).
  - `GET /app.js` ‚Üí the prebuilt browser bundle streamed from disk
    (`Content-Type: text/javascript`). Path resolution: `options.pageBundlePath`
    override (E2E injects a fixture) ‚Üí else `dist/page/app.js` resolved relative
    to `import.meta.url` (works from both `src/cli.ts` dev-run and `dist/cli.js`
    published-run). Missing bundle ‚Üí a fallback page naming
    `pnpm --filter @facet/quickstart build` (fail-safe; protocol routes still work).
  - `/agent/*` ‚Üí 404 at the wrapper (never proxied ‚ÄĒ see above).
  - everything else ‚Üí proxied to the internal facet server (headers/method
    forwarded per above).
- The browser bundle (`src/page/main.tsx`: `SseTransport` (same-origin `""` base)
  + `useFacet` + `StageRenderer` + `ChatDock`, with the Decision-2 `onAction`)
  is built at **package build time** by `tsup.config.ts` as a second config
  entry: `platform: "browser"`, `format: "iife"`, `noExternal: [/.*/]` (react,
  react-dom, and the workspace `@facet/client`/`@facet/react` are bundled in),
  **`define: { "process.env.NODE_ENV": '"production"' }`** (react's CJS entries
  branch on it; without the define the browser throws `process is not defined`
  ‚ÄĒ reviewer P1), minified, out to `dist/page/app.js`. The node entries
  (`index`, `cli`) build as today's bridge template does. Port busy on the
  public port ‚Üí exit 1 naming the port and `--port` (policy row S1-1).
- **The real bundle is executed by an automated check** (reviewer P1): WU-9's
  `e2e/bundle.test.ts` (run after `pnpm --filter @facet/quickstart build` as
  part of `/live-test` Tier 1) loads `dist/page/app.js` into a jsdom page with
  a `#root` element, asserts evaluation does not throw and the app mounts
  (root gains children), and asserts no bare `process.env.NODE_ENV` token
  survived in the bundle text.

### Decision 4 ‚ÄĒ provider interface: raw fetch, two adapters, deterministic key resolution

```ts
export interface ProviderMessage { readonly role: "user" | "assistant"; readonly content: string }
export interface ProviderTurn { readonly system: string; readonly messages: readonly ProviderMessage[] }
export interface QuickstartProvider {
  readonly name: "openai" | "anthropic";
  readonly model: string;
  /** One completion. Rejects on HTTP error, malformed body, or the 60s timeout. */
  generate(turn: ProviderTurn): Promise<string>;
}
```

- `createOpenAiProvider(apiKey, fetchImpl = fetch)` ‚Üí POST
  `https://api.openai.com/v1/chat/completions` (system as leading message);
  default model constant `DEFAULT_OPENAI_MODEL = "gpt-4o-mini"`.
- `createAnthropicProvider(apiKey, fetchImpl = fetch)` ‚Üí POST
  `https://api.anthropic.com/v1/messages` (`anthropic-version: 2023-06-01`,
  top-level `system`, explicit `max_tokens`); default model constant
  `DEFAULT_ANTHROPIC_MODEL = "claude-sonnet-4-5"`.
- **Raw fetch, no SDK deps** (AGENTS.md: no new dependency without a clear
  reason): each adapter is one POST endpoint + one JSON response shape; the
  official SDKs would add two heavyweight dependency trees to an npx-first
  package for zero capability. `fetchImpl` is injectable, which is what lets the
  DC-007 contract suite run the SAME behavior tests against both adapters with
  mocked HTTP and no network.
- **Timeout/retry:** `generate` aborts one attempt at 60s
  (`TURN_TIMEOUT_MS = 60_000`, AbortController). The agent's turn loop makes at
  most 2 attempts total (single retry), covering provider failure AND
  unusable output ‚ÄĒ the `generatePage` precedent (`generator.ts:111-131`).
- **Key/env resolution** (`resolveProvider(flags, env)`): explicit
  `--provider openai|anthropic` wins and REQUIRES its own key (missing ‚Üí error
  naming exactly `OPENAI_API_KEY` or `ANTHROPIC_API_KEY`); no flag ‚Üí
  `OPENAI_API_KEY` present ‚áí OpenAI (also when both are present ‚ÄĒ brief
  Decision Lock), else `ANTHROPIC_API_KEY` ‚áí Anthropic, else `null` (the CLI
  exits 1 with the DC-005 message). Keys are read from env only, never logged,
  never written (brief constraint).

### Decision 5 ‚ÄĒ 3-layer prompt + LLM-output parsing (salvage adapted, not imported)

`src/prompt.ts`:
- `buildSystem(guide: string)` = **layer ‚Ď†** (`STAGE_SPEC` from `@facet/core`
  (`spec.ts` ‚ÄĒ single source, extended with `collect` in WU-2) + a fixed output
  contract: *reply with ONE JSON object `{"say"?: string, "tree"?: <stage tree>}`,
  no prose/fences; reuse node ids when refining; use `field`s + a pressable box
  with `collect` for anything form-like*) + **layer ‚Ď°** (the deployer's guide
  markdown under a `PAGE BRIEF` heading). `DEFAULT_GUIDE` (a small built-in demo
  brief) lives here too.
- `buildTurnMessages(event, session, history)` = **layer ‚ĎĘ**: the last
  `HISTORY_TURNS = 20` sink `StoredEvent`s rendered as compact alternating
  user/assistant lines (visit / message text / action name+payload+fields; agent
  `say`s + a `(page updated)` marker), then the final user message = the current
  event (for `action` events including `event.fields`, so form submits are
  visible to the model) + `CURRENT STAGE: ${JSON.stringify(session.stage)}`.

`src/parse.ts`: `parseReply(text): { say?: string; tree?: unknown }` ‚ÄĒ the
fence-strip + string/escape-aware `balancedEnd` scan **adapted from
`apps/playground/src/generator.ts:16-75`** (copied on purpose: playground is
unpublished and nothing may depend on it ‚ÄĒ AGENTS.md dependency direction; a
future `/refactor-audit` may hoist a shared home). Acceptance: a
`{say?, tree?}` wrapper (tree tree-shaped or `{nodes}`-bearing), or a bare
tree-shaped object (treated as `{tree}`); nothing usable ‚Üí throw (caught by the
agent loop). `src/agent.ts` (`createQuickstartAgent`) then runs
`validateTree(reply.tree)` and only `stage.render`s a renderable result ‚ÄĒ
malformed output ‚áí stage untouched + apologetic `say`, process survives (DC-006,
mirroring `generator.ts:122` + `live-agent.ts` error posture).

### Decision 6 ‚ÄĒ stub brain: deterministic fixture agent, exported for the gate

`src/stub.ts` exports `createStubAgent(): FacetAgent` (via `defineAgent`), fully
deterministic, zero network:

- `visit` ‚Üí `stage.render(STUB_TREE)`: a hero text; box `signup` containing
  `field(name:"name")` + `field(name:"email")`; a pressable box
  `onPress: {kind:"agent", name:"submit", collect:"signup"}`; plus
  `screens: {home, about}` + a navigate box ‚ÄĒ so the Tier-1 E2E exercises
  collect, screens, and patching in one tree.
- `message` ‚Üí a deterministic patch upserting text node `stub-echo` with
  `echo: <text>` + `say "stub: <text>"`.
- `action` ‚Üí `say` echoing `name` and the sorted `key=value` pairs of
  `event.fields ?? {}` ‚ÄĒ the assertion hook that proves field delivery
  end-to-end (DC-008).

Also the `--stub` keyless demo mode (DC-005's guidance target).

### Decision 7 ‚ÄĒ `/live-test`: repo skill + vitest E2E, 3 tiers

- **Tier 1 (always, blocking), two parts:**
  **(1a)** `packages/quickstart/src/quickstart.e2e.test.ts` ‚ÄĒ inside the root
  vitest glob, so it also runs in plain `pnpm test`. Reuses the
  `server.test.ts` harness pattern (random-port boot, fetch POST clients, SSE
  frame readers): boots `startQuickstart` with `createStubAgent()` + an injected
  page-bundle fixture, then asserts GET `/` shell, GET `/app.js` static route,
  proxied `/health`, SSE `/stream` through the proxy, **reconnect with
  `Last-Event-ID` resuming through the proxy** (request-header forwarding
  pinned), **`/agent/stream` answering 404 at the wrapper** (agent channel not
  exposed), visit snapshot (signup form + screens present), chat patch
  (`stub-echo`), collect press with `fields` echoed in the `say`, and rapid
  double-message ordering. No key, no network, deterministic (DC-008).
  **(1b)** `packages/quickstart/e2e/bundle.test.ts` ‚ÄĒ run via the e2e vitest
  config AFTER `pnpm --filter @facet/quickstart build`: jsdom-executes the REAL
  `dist/page/app.js` (mount asserted, no throw, no bare `process.env` token in
  the bundle text). The test installs a minimal no-op `EventSource` stub on the
  jsdom global BEFORE evaluating the bundle (jsdom/Node ship no `EventSource`;
  `SseTransport` constructs one on mount ‚ÄĒ browsers have it, the harness must
  stub it; no fetch stub needed, sends queue until `onopen`). This closes the
  "fixture bundle only" gap: the page the adopter actually loads is executed
  by the gate.
  **Honest seam note (recorded, accepted):** 1a proves
  server-guard‚Üíruntime‚Üíagent‚Üísink‚ÜíSSE with the harness posting `fields`
  itself; the renderer's production of those fields is proven at unit level
  (WU-3 jsdom) and the page glue by 1b's mount + the final-gate manual DC-004
  run. No single automated test drives a real browser click end-to-end ‚ÄĒ a
  known, accepted layering (recorded in WU-9 handoff).
- **Tier 2 / Tier 3 (smoke):** `packages/quickstart/e2e/smoke.test.ts` ‚ÄĒ OUTSIDE
  the root glob (root include is `packages/**/src/**`), run only via its own
  `packages/quickstart/e2e/vitest.config.ts` (`testTimeout: 120_000`). One real
  turn against the resolved default provider with `DEFAULT_GUIDE`; **loose
  assertions only**: a patch frame arrives whose tree passes `validateTree` and
  is renderable; no crash; never content matching. `FACET_SMOKE_PROVIDERS=both`
  makes it run both adapters and treat a missing key as a hard failure (Tier 3).
- **Skill (`.claude/skills/live-test/SKILL.md`), tier detection (robust ‚ÄĒ
  reviewer P2):** candidate diff = union of (a) uncommitted changes
  (`git status --porcelain`) and (b) committed delta vs the base:
  `git merge-base HEAD origin/main` first, falling back to `main`, falling
  back to `HEAD~1`. If every base candidate errors (shallow clone, detached
  HEAD, first commit) **or** the merge-base equals HEAD with a clean tree
  while the gate is invoked for a change (can't tell what changed), the skill
  **assumes quickstart-touched** ‚ÄĒ conservative: over-verifying costs one
  smoke turn; under-verifying defeats DC-009. *quickstart-touched* ‚áĒ any
  candidate path starts with `packages/quickstart/`.
  Tier 1 (1a + 1b) runs always and blocks on failure. Tier 2 runs when
  touched: no key present ‚áí report **FAIL** ("SKIPPED = FAIL for
  quickstart-touching changes", DC-009); not touched ‚áí Tier 2 optional, skip =
  OK with reason. Tier 3 runs pre-merge/release on request. AGENTS.md's DoD
  chain gains `/live-test` after `/code-review` (WU-10).

### Decision 9 ‚ÄĒ CLI flags (exact contract)

`facet-quickstart [--guide <path>] [--port <n>] [--provider openai|anthropic] [--stub] [--agent-id <id>]`

- `--guide` default `./facet.md`; an **explicitly passed** path that doesn't
  exist ‚Üí exit 1 naming the path (DC-005); the *default* path absent ‚Üí
  `DEFAULT_GUIDE` fallback (Decision Lock).
- `--port` default `5292`; busy ‚Üí exit 1 naming the port + `--port`.
- `--provider` per Decision 4; `--stub` skips key resolution entirely and wires
  `createStubAgent()`; `--agent-id` default `"quickstart"`.
- No key and no `--stub` ‚Üí exit 1:
  `"No provider key found. Set OPENAI_API_KEY or ANTHROPIC_API_KEY, or run with --stub for a keyless look around."`
- On boot: print the link and the resolved provider/model. `cli.ts` keeps the
  shebang + a thin `main()`; the arg-parse/key-resolution/boot logic is an
  exported `runCli(argv, env): Promise<number>` so `cli.test.ts` drives DC-005
  in-process (no child spawn; deterministic exit codes + messages).

### WU sizing rationale

10 WUs: the ‚Č§5-file cap and one-package-per-WU boundaries force the split ‚ÄĒ
WU-1 (core contract) must land alone first because everything narrows against
its types; WU-2/3/4/5 then fan out in parallel on disjoint file sets
(core-spec / react+playground / server / quickstart-scaffold); the quickstart
tail (WU-6 prompt+parse ‚Üí WU-7 agent+stub ‚Üí WU-8 bin+page+barrel ‚Üí WU-9 gate)
is inherently serial because each layer consumes the previous one's exports
(and the barrel `index.ts` must be written exactly once, in the last code WU);
WU-10 (docs+changeset) closes after the gate exists so AGENTS.md documents a
runnable `/live-test`.
