### Technical approach

Narrow `FacetComposition` to a required concrete native subtree:
`{name, metadata: {description,...}, root, nodes: Record<NodeId,FacetNode>}`.
Reuse the existing native sanitizer/topology bounds while retaining fragment
root semantics. Delete every slot/reference/graph/expansion path and its public
surface, including `Stage.useComposition`. Remove expansion fill hooks from the
brick registry. Recast catalog `compositions` as an exposure allow-policy only
and change the authoring order to `component -> primitive`.

In `@facet/agent-tools`, replace `use_composition` with a distinct read-only
`get_composition({name})`. A small public
`selectCompositionReferences(compositions: readonly unknown[], catalog?: unknown): readonly FacetComposition[]`
helper in `composition-references.ts` solely owns validation, first-valid
dedupe, exposure filtering, and detached deep-freezing for the snapshot used by
both prompt indexing and lookup; it stops after 128 selected references so the
largest valid file collection cannot overflow the smallest reference-agent
context profile, and it has no runtime/store imports or hardcoded names. Prompt
output is only name + `metadata.description`. Known lookup returns
the exact serialized composition in the normal structured observation with zero stage effects;
unknown/disallowed/malformed requests reject fail-safe. The lookup opts into an
exact-data observation mode and the reference transcript preserves the complete
result past the generic 2K and 4K/8K/12K caps. The normal total-context hard
limit remains: an over-budget whole result stops `context_limit`, never partial.

Quickstart continues to call `loadAssets` once, and reference-agent clones that
loaded list/catalog once. The same immutable snapshot feeds both prompt and
executor. The deterministic E2E changes from “expand and patch” to “read, then
author native nodes”; composition JSON stays provider-side and never enters the
browser shell/SSE protocol.

## Decision Locks

1. **Hard cutover:** no aliases, deprecated names, compatibility parser, dual
   tools, or runtime expansion fallback.
2. **Required description location:** `metadata` and `metadata.description` are
   required; top-level `description` is removed.
3. **Native vocabulary:** current `FacetNode` is the closed dataset vocabulary.
   This change does not perform the later box-only node-model migration.
4. **Fragment root:** any retained native leaf/container may be root.
5. **Marker semantics:** none. Marker-looking strings are literal; official
   datasets/fixtures/docs contain none.
6. **Prompt/read source:** one validated, deduped, catalog-filtered immutable
   snapshot; no second list. The public selector signature is exactly
   `selectCompositionReferences(compositions: readonly unknown[], catalog?: unknown): readonly FacetComposition[]`.
   It is the sole owner of composition exposure filtering and imports only core
   validation/types. Runtime-loaded input is already unique; for a direct
   duplicate array, the first valid occurrence wins. Omitted catalog exposes all
   valid references within the deterministic 128-reference exposure cap; a
   supplied malformed catalog fails closed to zero. The
   result is a newly allocated, deeply frozen array whose nested composition
   objects retain no caller-owned references.
7. **Catalog:** `FacetCatalog.compositions` stays as exposure policy;
   `CatalogUsageOrder` becomes `['component','primitive']`.
8. **Tool:** only `get_composition`; exact input `{name}`; no params/slots/at/
   target/override/patch/insertion.
9. **Success observation:** standard structured `no_stage_change` envelope;
   exact complete serialized composition in `data`; `JSON.parse(data)` is
   deep-equal to the selected validated composition with no field loss; zero
   messages/patches/changed IDs and identical shadow. The public
   `formatAgentToolObservation(...)` remains capped and gains no bypass option;
    only a package-private, role-specific composition observation formatter used
    by the asset executor may create this exception.
10. **Unknown/disallowed:** `invalid_composition` rejection (not an ok result),
    no throw or mutation.
11. **Exact-result size:** bypass per-observation truncation for this tool only.
    If the whole transcript exceeds provider context, stop `context_limit` before
    the next provider call; never slice or return `{"truncated":true}`. Even
    when `minRecentStepsVerbatim` is zero, the newest composition-read group is
    pinned verbatim until its first provider handoff; if it cannot fit, stop
    `context_limit` rather than summarize it.
12. **Persistence:** File/Postgres remain opaque raw stores; no SQL/schema
    change. `loadAssets` is the semantic gate.
13. **Search scope:** shipping code/tests/docs/fixtures/package READMEs/current
    changesets must have no legacy surface. Historical `specs/**` are archival;
    the four pre-existing untracked display-leaf spec files remain untouched.
14. **Approval boundary:** no worktree, implementation, branch, push, PR, or
    merge before owner approval of this spec/manifest.

## Fail-safe and Boundary Checklist

- Missing/empty/malformed name or extra tool fields: `invalid_input`, unchanged
  buffer/shadow.
- Unknown/catalog-disallowed name: bounded `invalid_composition`, unchanged
  buffer/shadow.
- Hostile/untrusted composition documents: `validateComposition` never throws;
  unusable documents never enter the exposed snapshot.
- Native raw HTML/JS/CSS/fetch/query/resolver/invalid token/overlay: existing
  sanitizer drops/refuses it; lookup returns only the sanitized object.
- Dangling children/child cycles/depth/node-cap: existing composition fragment
  topology policy applies; no graph-of-compositions check exists.
- Leaf root: accepted when the native root survives.
- Repeated/concurrent reads: deterministic and non-mutating.
- Pending buffer edits: unchanged by read and resolve afterward as before.
- Result >2K/>4K: complete exact serialized data reaches the next provider call,
  including when configured recent-step retention is zero.
- Result exceeds total context: provider is not called with a partial value;
  existing fail-safe `context_limit` termination.
- Browser/reconnect: no composition JSON/provenance/global/SSE frame.
- Offline/provider failure after a read: existing bounded failure response, no
  partial stage write; deterministic loop/E2E regressions remain green.
- Rapid/racing visitor events: existing runtime serialization and per-visitor
  stage isolation remain unchanged; the read adds no writer or version path.
