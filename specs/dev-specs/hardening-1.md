## Fail-safe & Boundary Checklist

- Malformed / empty / null input: strengthened everywhere touched — duplicate
  ids deduped (H-01), non-primitive payloads filtered/rejected (H-02), field
  props coerced (H-14), prototype-key tokens resolve to nothing (H-15),
  wrong-shape session files ignored with a logged error (H-11), oversized
  bodies rejected 400 (H-16), malformed persistent-driver events settle alone
  (H-06 pins this). `applyPatch` remains the one deliberate thrower (RFC
  contract), absorbed by the runtime salvage path.
- Deeply nested / cyclic tree: unchanged behavior, now single-sourced —
  `MAX_DEPTH` exported from `@facet/core` (H-04); verified today's `>=`
  (validate, parent-depth) and `>` (renderer, self-depth) both keep exactly
  depth ≤ 100, so consolidation is behavior-neutral and pinned by the existing
  depth tests.
- Offline agent / rapid events / racing agent turn: reconnect rehydrate ordered
  before live fan-out (H-03); visitor POSTs serialized per transport (H-17);
  sink records serialized per session (H-19); terminal 403/409 stops the
  reconnect loop loudly instead of silent forever-retry (H-18); persistent
  driver dead-guard/settleAll pinned by new tests (H-06).
