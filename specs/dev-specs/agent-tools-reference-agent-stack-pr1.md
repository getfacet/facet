## Invariant Fit Audit (Facet - REQUIRED)

| # | Invariant | Touched? | Safe design |
|---|---|---|---|
| 1 | Responsibility boundary (no backend/domain in Facet) | Yes | `@facet/agent-tools` only provides stage mutation, say, inspection, and shadow folding helpers. It has no provider, fetch, database, business API, or domain tool code. |
| 2 | Mechanism vs policy | Yes | `@facet/agent-tools` is mechanism only. Reference-agent remains one policy implementation and is not required by external agent authors. |
| 3 | Fail-safe | Yes | Invalid model tool input returns typed observations and emits no patch. Successful patch messages are folded through `foldPatchIntoStage`, which context evidence shows is pure and never throws. |
| 4 | Declarative + tokens only | No new vocabulary | Tool specs and executor only accept existing `FacetTree` / `FacetNode` shapes and route through existing validation/fold behavior. |
| 5 | Flow-only safety | No | No layout vocabulary, overlay, or positioning change. |
| 6 | Two-writers coherence | Yes | Within a provider turn, agent-tools maintains a local shadow by folding emitted patches in order. Runtime remains authoritative after yielded batches; PR2 will seed each turn from `session.stage`. |
| 7 | Backend via agent tools, never client fetch | No | No client fetch or backend/domain operation is added. |

## Fail-safe & Boundary Checklist

- Malformed / empty / null input: executor returns `status: "error"` observation,
  emits no patch, leaves shadow unchanged.
- Deeply nested / cyclic tree: full-tree render goes through `validateTree` /
  `foldPatchIntoStage`; issues are summarized and bounded.
- Invalid parent / non-box parent: executor returns typed error and emits no
  child-link patch.
- Oversized patch or stamp expansion: executor refuses before emitting partial
  patches when the emitted batch would exceed `MAX_PATCH_OPS`.
- Offline agent / provider failure: unchanged in PR1; reference-agent keeps
  existing fallback behavior. PR2 owns richer retry/trace policy.
- Rapid events / racing agent turn: unchanged in PR1; runtime serial queue stays
  authoritative. Local shadow only orders tool calls within one model turn.
- Path migration boundary: package names and public import specifiers stay
  stable; only repository directories and local paths change.
- Coverage mapping:
  - WU-8 `executor.test.ts` covers malformed/null input, invalid parent/non-box
    parent, bounded inspection, render/fold issue summaries, and unchanged
    shadow on errors.
  - WU-9 `agent.test.ts` keeps provider failure, max-step loop, and
    reference-agent compatibility coverage green after the executor move.
  - Final `/live-test` covers real quickstart boot, SSE delivery, and racing-path
    regressions on the live-link surface.
