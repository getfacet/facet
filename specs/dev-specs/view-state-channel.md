## Overview
- Feature slug: `view-state-channel`
- Intake brief: `specs/feature-intake/view-state-channel.md`
- Context evidence: `specs/context/view-state-channel.md`
- Affected `@facet/*` packages: `@facet/core`, `@facet/server`, `@facet/react`,
  `@facet/client`, `@facet/agent-tools`, `@facet/reference-agent`,
  `@facet/ag-ui` (+ host wiring in `@facet/quickstart` and `apps/playground`,
  unpublished).
- One-paragraph technical approach: Ride a bounded, browser-owned **view
  snapshot** (`screen`, `toggled`, `viewport`, `scheme`) on the three
  already-forwarded `ClientEvent` kinds (visit/message/tap-with-action). Add the
  optional `view?: ViewSnapshot` field to `ClientEvent`/`CollectedEvent` and a
  single pure, node-free `sanitizeView` (caps + closed enums) in `@facet/core`,
  reusing the `MAX_FIELD_*`/`sanitizeActionPayload` precedent so client and
  server can't drift. `@facet/react`'s `StageRenderer` (the sole owner of
  `currentScreen`/`visibilityOverrides`) publishes its live snapshot up through a
  read-only callback â€” exactly like `fields` rides `onAction` â€” plus new
  `matchMedia`-based viewport/scheme detection that only mutates in-memory state
  and never sends. The **host** composes `view` onto the outgoing event (never
  `@facet/client`, which stays framework-neutral and only contributes a
  per-agent-link `localStorage` persistence helper for the revisit `visit`
  snapshot). The `@facet/server` `/event` boundary **clamps** `view` via the core
  sanitizer (drop/clamp, never reject the event); `@facet/ag-ui`'s input
  normalizer mirrors that clamp so adapter-path agents aren't blind. The agent
  surface renders one inert prompt line in `describeEvent`; `view` never reaches
  any patch/fold/executor path (server stays the only stage writer, no
  auto-restore).

## Fail-safe & Boundary Checklist
- **Malformed / empty / null `view`**: `sanitizeView(unknown)` returns a cleaned
  `ViewSnapshot` or `undefined`; the `/event` clamp replaces `event.view` with the
  cleaned value (or omits it) and the event still processes. `null`/non-object â†’
  `undefined`. Never a 400 for `view` reasons.
- **Oversized / over-cap `view`**: `screen` and each toggled node-id key
  length-capped at `MAX_FIELD_VALUE_CHARS` (dropped if over); `toggled` capped at
  `MAX_VIEW_TOGGLED_KEYS` (drop-oldest beyond cap); toggled values not
  `"shown"|"hidden"` dropped; `viewport`/`scheme` outside the closed enums dropped.
  The 5 MiB `MAX_BODY_BYTES` cap still bounds the whole body upstream.
- **Deeply nested / cyclic `view`**: `sanitizeView` reads only known scalar/enum
  fields and a flat stringâ†’enum `toggled` record; it never recurses into nested
  objects, so a nested/cyclic payload yields at most a cleaned flat snapshot (its
  non-conforming entries dropped) â€” no traversal, no throw.
- **Storage unavailable / throwing** (SSR, sandboxed iframe): `view-storage.ts`
  guards `typeof localStorage === "undefined"` and wraps get/set in try/catch
  (visitor.ts precedent) â†’ live in-memory snapshot only; no crash, no console spam.
- **Offline agent / rapid events / racing agent turn**: each event snapshots at its
  own send moment (same as `fields`); stale-by-one is acceptable. No new lane,
  ordering, or race surface â€” `view` is inert data on an existing event; the
  server-as-sole-stage-writer and existing `seq` ordering are untouched.
- **Older client (no `view`)**: absent `view` = today's behavior everywhere;
  `describeEvent` omits the line; ag-ui/server pass the event through unchanged.
