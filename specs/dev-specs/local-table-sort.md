## Overview
- Feature slug: `local-table-sort`
- Intake brief: `specs/feature-intake/local-table-sort.md`
- Context evidence: `specs/context/local-table-sort.md`
- Affected `@facet/*` packages: `@facet/core`, `@facet/react`, `@facet/agent-tools`,
  `@facet/reference-agent`
- One-paragraph technical approach:
  A `table` column opts into sorting through an additive, validated
  `TableColumn.sortable?: boolean` (drop-with-issue on a non-boolean). The visitor
  clicking a sortable header cycles `asc â†’ desc â†’ unsorted` as a **pure render-time
  reorder**: `StageRenderer` gains a THIRD browser-private view-state holder â€” a
  `Map<NodeId, { column, direction }>` alongside `currentScreen` /
  `visibilityOverrides` â€” mutated only in a new header-click setter, threaded into
  `renderTable` through `BrickRenderContext`, and applied by a new renderer-owned,
  closed, total, stable comparator (`table-sort.ts`) to the already-resolved+capped
  `rows` immediately after `resolveNodeData`. The renderer NEVER writes back to
  `context.data` / `node.rows` and NEVER caches the sorted array, so a later server
  `data` patch re-resolves+re-sorts automatically (two-writers coherence, server sole
  writer). The header click DEVIATES from navigate/toggle: it fires **no**
  `recordLocalTap`/`onRecord` and **no** transport call â€” the sort rides ONLY the
  existing `view` snapshot. `@facet/core` `ViewSnapshot` gains an optional, capped,
  sanitized `sort` map plus a closed `SortDirection` enum; `captureViewSnapshot`
  emits it and the publish `useEffect` dependency array gains the sort holder. The
  LLM-facing prompt-kit guidance and the reference-agent `describeView` line learn to
  mention the per-column opt-in and the current `view.sort`.

## Fail-safe & Boundary Checklist
- **Malformed / empty / null input:** non-boolean `sortable` â†’ dropped with an issue,
  column still renders (WU-1). A `view.sort` value that is not a plain object, or an
  entry whose column is not a string / direction outside the closed enum, is dropped
  by `sanitizeView`; nothing valid remaining â‡’ `sort` omitted (WU-2). An empty rows
  array or absent cells sort through the total comparator without throwing (WU-3a).
- **Deeply nested / cyclic tree:** `sanitizeView` reads only the known flat `sort` map
  and never recurses (mirrors `toggled`); a cyclic/hostile payload yields at most a
  cleaned flat snapshot or `undefined` (WU-2). The comparator reads scalar cells only.
- **Offline agent / rapid events / racing agent turn:** rapid header clicks cycle
  deterministically, last click wins (React state, single setter). A sort change with
  no agent connected simply mutates local state and re-renders â€” nothing to send. A
  server `data` patch arriving while sorted re-resolves+re-sorts on the next render;
  the sort holder is untouched by the patch path (DC-003). An inert previous-screen
  clone wires NO live sort dispatcher (RISK-INV-5), so no second view-state writer
  exists mid-transition.
