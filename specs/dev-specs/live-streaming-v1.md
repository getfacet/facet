## Overview
- Feature slug: `live-streaming-v1`
- Intake brief: `specs/feature-intake/live-streaming-v1.md`
- Context evidence: `specs/context/live-streaming-v1.md`
- Affected `@facet/*` packages: `@facet/core`, `@facet/agent`, `@facet/runtime`,
  `@facet/server`, `@facet/agent-client`, `@facet/bridge`, `@facet/quickstart`.
  (The last two were added by the context pass per RISK-API-1 â€” they call/await the
  agent and would fail `pnpm typecheck` once the return union widens.)
- One-paragraph technical approach:

  Widen `FacetAgent`'s return surface from `array | Promise<array>` to a **union**
  that also admits `AsyncIterable<readonly ServerMessage[]>` (RISK-API-3: additive
  widening, not a hard break â€” every existing array/Promise producer stays valid).
  Add ONE shared normalize pair in `@facet/core`: `iterateAgentResult()` (any of
  the three forms â†’ an async iterable of batches, used by the runtime to drive a
  turn) and `collectMessages()` (any of the three â†’ one flat array, used by the two
  non-streaming boundaries). The runtime's turn driver stops doing one apply+persist
  at turn end and instead **iterates the agent's batches inside the existing
  per-visitor serial lane**, folding **each batch once** (identical to today's
  per-turn coalescing, just at batch granularity), saving, and delivering each batch
  synchronously via an injected frame sink **before pulling the next** â€” while
  keeping ONE Sink record per turn (accumulated), ONE seed frame (on the first
  non-empty batch only), and ONE aggregate `agentMutated` (OR across batches). The
  server passes the per-batch deliver callback into `runtime.handle` (delivery stays
  inside the lane task); the remote agent-client boundary collapses to a single
  batch (it cannot stream â€” explicit v1 non-goal). `@facet/agent` gains
  `defineStreamingAgent` (an async-generator producer that flushes the `Stage` per
  step) and `Stage.flush()` becomes drain-and-reset so repeated flushes yield
  deltas; `@facet/quickstart` uses it to yield one batch per provider step, emitting
  only **referentially-closed** batches (forward child-refs held until their target
  node exists) so per-batch folding can never orphan a forward reference.

## Fail-safe & Boundary Checklist
- **Malformed / empty / null input:** an empty batch (`[]` or all-`test` ops) â†’
  NO frame delivered and no seed loss (seed rides the next non-empty batch);
  `foldPatchIntoStage` salvages malformed patches per op (unchanged). A non-array
  yielded value is impossible under the typed union; `iterateAgentResult` only
  yields the producer's own arrays.
- **Deeply nested / cyclic tree:** unchanged â€” `validateTree`/`foldPatchIntoStage`
  cap depth and prune dangling children per batch exactly as per-turn today.
- **Forward child-ref across batches (RISK-INV-1):** within a batch, forward refs
  are safe (concatenate-then-fold-once). Across batches, the yield boundary is
  defined as **referentially closed** in the parentâ†’child direction: producers must
  not emit a child-ref op in an earlier batch than its target-node op. Quickstart
  enforces this by construction (`append_node` is atomic; `set_node`/`render_page`
  forward-ref ops are held until their target ids exist, then flushed â€” never split).
- **Offline agent / rapid events / racing agent turn:** the remote agent-client
  boundary collapses the union to ONE batch (cannot stream â€” v1 non-goal); the
  `/event` POST still acks 202 immediately; a same-visitor late/resume apply
  (`applyMessages`) stays single-frame and enqueues behind the streaming lane task;
  different visitors remain fully parallel (separate lanes).
