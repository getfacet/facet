# Feature Intake Brief: Landing-Grade Vocabulary (v1)

> Generated by `/feature-intake`. Plain-language, testable, invariant-checked.

## Goal

- An agent can compose a **landing-page-grade** page — an Apple/Browserbase-style
  full-height hero with large display type over a background image, plus richly
  treated sections (dark bands, gradients, sticky nav, highlighted headlines) —
  using only Facet's closed token/brick vocabulary. No pixels, no raw CSS, no
  absolute positioning: the expressiveness comes from a **deliberately widened
  token vocabulary**, not an escape hatch.

## Current Problem

- Facet's vocabulary is **dashboard-scale**. `Sizing` is only `auto|full` (no
  height concept), `FontSize` tops out at `3xl`, there is no way to place content
  over a background image, no sticky, no gradient, no dark-section, no display
  typography. So a Facet document renders like an app dashboard and **cannot
  express a marketing/landing page** — the "living, per-visitor page an agent
  owns" story falls flat at first paint.
- Affected: the **agent author** (can't produce a landing-grade page no matter
  how it prompts) and the **end-user** (sees a dashboard where a hero belongs).

## User Scenario

### Who
- The **agent brain** authoring a stage (emits nodes + token styles); the
  **end-user** views the rendered landing page; the **operator** may re-skin the
  new tokens via a theme.

### Trigger
- The agent draws a landing/marketing page: a hero, then content sections.

### Steps
1. The agent emits a `box` with `backdrop: "<mediaNodeId>"`, `minHeight:"screen"`,
   `justify:"center"`, holding a `text` headline (`size:"6xl"`, `tracking:"tight"`,
   `highlight:"accent"`) and a CTA `button` — a full-height hero with the copy
   over the image.
2. The renderer paints the referenced media as the box's background LAYER and the
   flow children on top, applying a readability **scrim** token; children still
   stack in normal flow (no overlap, no z-index).
3. Below, the agent emits sections: one with `scheme:"dark"` (a dark band), one
   with a `gradient` background, a `sticky` sub-nav, a constrained text column via
   a container `maxWidth` token.
4. The rendered page reads as a landing page. An operator theme may override the
   concrete values of any new token group (heights, gradients, display sizes,
   scrim) without touching the tree.
5. If a token value is unknown or a `backdrop` names a missing/invalid media
   node, the renderer degrades (no backdrop / default token) — never throws.

## Input/Output Examples

### Example 1 (Normal — hero)
- Input: `{ type:"box", backdrop:"hero-img", style:{ minHeight:"screen",
  justify:"center", align:"center", backdropScrim:"dark" },
  children:["headline","cta"] }` + `{ type:"text", value:"Give your agents the
  whole web.", style:{ size:"6xl", weight:"bold", tracking:"tight",
  highlight:"accent" } }` + `{ type:"media", id:"hero-img", kind:"image",
  src:"…" }`.
- Output: a full-viewport-height hero; the image fills the background behind a
  dark scrim; the large headline (with an accent highlight behind the text) and
  CTA sit centered on top, in normal flow.

### Example 2 (Edge — dangling / unknown)
- Input: a box with `backdrop:"ghost"` (no such media node), `minHeight:"galaxy"`
  (unknown), a text `size:"9xl"` (unknown), `gradient:"plaid"` (unknown).
- Output: the box renders with NO backdrop, default min-height, default font
  size, no gradient — `validateTree` drops the unknown token values with issues;
  nothing throws; the page still renders its content plainly.

### Example 3 (Edge — backdrop target is not a usable media node)
- Input: `backdrop` names a `text` node, or a media node with an unsafe/blank
  `src`.
- Output: no background layer is painted (the existing safe-`src` media rule
  still gates it); the box renders its flow children normally; no throw.

## Constraints (What Not to Do)

- **No raw CSS / pixels / hex.** Every new capability is a CLOSED token set:
  heights, container widths, display sizes, tracking, leading, gradients, scrim,
  scheme, highlight are all named tokens whose concrete values live in a theme.
  NO raw CSS gradient strings, NO arbitrary length/color values in the tree.
- **No general overlap / z-index escape hatch.** `backdrop` yields EXACTLY two
  renderer-controlled layers (background media behind, flow content in front).
  Children never get absolute positioning; they cannot overlap each other or
  escape the box. `sticky` is `position:sticky` (flow-compatible), NOT absolute.
- **No new brick kind for the hero.** The hero is a `box` + `backdrop`
  attribute (a general capability), not a dedicated `hero` component — this keeps
  the primitive base composable (background cards, banners get it for free).
- **No motion, icons, copy-action, or effect tokens in this bundle** — declarative
  motion (reveal/parallax), an icon vocabulary, a `copy` action kind, and
  pointer-reactive effect tokens are explicitly OUT (later bundles / the renderer
  extension API).
- **No backend, no local execution, no new client writer.** Pure additive
  presentation vocabulary; nothing fetches, computes, or writes stage state.

## Policy & Edge Cases

| Scenario Step | Failure Policy | Boundary/Validation | Concurrency / Duplicate Trigger |
|---|---|---|---|
| 1 author tokens | unknown token value → dropped with an issue, node keeps rendering | every new token is a closed-set membership check in `validateTree` (like existing tokens) | pure validation; no state |
| 1 `backdrop` ref | names a missing node / non-media / unsafe-`src` media → no backdrop painted, flow content renders | `backdrop` is a node-id string; renderer resolves it read-only and reuses the existing safe-media-`src` gate | no writer; render-time lookup |
| 2 scrim/layering | no scrim token → renderer default; backdrop present but children empty → just the image | scrim is a closed token; layering is renderer-owned (two layers, no z-index) | renderer-owned, deterministic |
| 3 dark-section (`scheme`) | unknown scheme → default (light) | `scheme` flips color-token resolution for that subtree only; bounded, renderer-owned | pure subtree resolution |
| 3 gradient | unknown gradient token → no gradient (falls back to `bg`/none) | gradient is a closed token set; concrete stops live in the theme, never in the tree | — |
| 3 sticky | sticky in a non-scrolling context → renders as normal flow (no-op), never breaks layout | `position:sticky` is bounded by the renderer; no offset authored by the agent | — |
| theme override | malformed theme value for a new group → existing `validateTheme` clamp/drop path | new token groups plug into the existing theme override + WCAG/clamp machinery | — |

## Done Criteria (Testable)

- [ ] DC-001 (Happy path — hero renders): a `box` with `backdrop` + `minHeight`
  renders the referenced media as a background layer with the flow children on
  top and a scrim; the display-size headline + highlight render. → Verify:
  `@facet/react` render test (static + jsdom) + `validateTree` accepts the new
  tokens/attribute.
- [ ] DC-002 (New token groups validated): each new token (heights, container
  maxWidth, `4xl/5xl/6xl`, tracking, leading, gradient, scrim, scheme, highlight)
  is accepted when valid and DROPPED with an issue when unknown; `validateTree`
  never throws. → Verify: `validate.test.ts` membership + hostile cases.
- [ ] DC-003 (Fail-safe layering): a dangling `backdrop`, a `backdrop` pointing
  at a non-media/unsafe-`src` node, unknown token values, and a deeply nested
  tree all render without a backdrop/with defaults and NEVER throw. → Verify:
  fail-safe render + validate boundary tests.
- [ ] DC-004 (Flow-only preserved): `backdrop` produces exactly two layers and
  children stay in normal flow — no absolute positioning, no z-index escape, no
  child overlaps a sibling or escapes the box; `sticky` stays in flow. → Verify:
  renderer layout-contract test (structural: no `position:absolute`, bounded
  layers).
- [ ] DC-005 (Theme override): an operator theme overriding a new token group
  (e.g. gradient stops, display sizes, scrim) re-skins the page with no tree
  change; malformed theme values go through the existing clamp/drop path. →
  Verify: `theme.test.ts` + `@facet/assets` DEFAULT_THEME resolution.
- [ ] DC-006 (Back-compat): existing trees (no new tokens, no `backdrop`) render
  byte-identically; all new node/style fields are additive optionals. → Verify:
  existing render + validate suites stay green + explicit no-new-token case.
- [ ] DC-007 (Agent vocabulary taught): `STAGE_SPEC` documents the new tokens +
  `backdrop` (closed sets, names only, no raw CSS) so the LLM can author a hero. →
  Verify: `spec.test.ts` assertions on STAGE_SPEC content.

## Invariant Fit (Facet-specific — REQUIRED)

| # | Invariant | Status | Note / mitigation |
|---|---|---|---|
| 1 | Responsibility boundary (UI-out/UI-in only; no backend) | OK | Pure presentation vocabulary; nothing fetches or computes. `backdrop` references an in-tree media node (already agent-authored), not a URL data source. |
| 2 | Mechanism vs policy | OK | Facet provides the token vocabulary; the agent decides the design. Concrete values live in the theme (operator), never hard-coded into the framework's policy. |
| 3 | Fail-safe | TOUCHES (mitigation) | Many new validated token groups + a new `backdrop` node reference. All are closed-set membership checks that DROP unknowns with an issue; dangling/unsafe `backdrop` → no background layer; never throws (mirrors existing token/media fail-safe). |
| 4 | Declarative + tokens only | TOUCHES (mitigation) | This is the whole feature. Every capability is a CLOSED token (heights, widths, display sizes, tracking, leading, gradient, scrim, scheme, highlight) — NO raw CSS gradient strings, lengths, or hex in the tree. Concrete values resolve in a theme, exactly like existing tokens. |
| 5 | Flow-only safety | TOUCHES (mitigation) | `backdrop` is the one overlap. It is renderer-owned and bounded to EXACTLY two layers (background media, flow content) — no z-index, children never overlap or escape, no absolute positioning is ever emitted. `sticky` is `position:sticky` (stays in flow). Not a general overlay/escape hatch. |
| 6 | Two-writers coherence | OK | No local execution added; the renderer only reads tokens + resolves the `backdrop` node id read-only. Server stays sole stage writer. |
| 7 | Backend via agent, never client fetch | OK | No client fetch; `backdrop` media uses the existing safe-`src` media path (already gated), no new network capability. |

## Public API / Package Surface

| `@facet/*` package | Change | Additive or Breaking |
|---|---|---|
| `@facet/core` | new token groups in `tokens.ts` (heights, container maxWidth, `4xl/5xl/6xl` FontSizes, tracking, leading, gradient, scrim, scheme, highlight); `BoxStyle` gains `minHeight`/`maxWidth`/`sticky`/`gradient`/`backdropScrim`/`scheme`; `BoxNode` gains `backdrop?: NodeId`; `TextStyle` gains `tracking`/`leading`/`highlight`; `validateTree` validates them; `STAGE_SPEC` teaches them | additive |
| `@facet/react` | `boxStyle`/`textStyle` map the new tokens to CSS; renderer paints the `backdrop` layer + scrim; `scheme` subtree color inversion; sticky/gradient/display type/highlight rendering | additive |
| `@facet/assets` | `DEFAULT_THEME` gains concrete values for every new token group | additive |
| `@facet/core` `theme.ts` | new token groups are overridable via the existing theme machinery (allowlist + clamp + WCAG) | additive |
| runtime/server/client/ag-ui/agent | none — tokens/attributes ride the tree/patches through the existing pure `applyPatch`; no protocol change | — |
