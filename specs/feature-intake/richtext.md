# Feature Intake Brief: Rich Text Brick (PR-2)

> Generated by `/feature-intake`. Part of the node-model restructure (design
> LOCKED; PR-0 registry + PR-1 binding-enablers + PR-1.5 node-mixins MERGED,
> main=35d8c55). NEW feature. Extensibility deliberately front-loaded (owner
> instruction): closed shapes designed to grow additively toward a
> Markdown-complete rich text, without a breaking reshape.

## Goal

- Add a closed, fail-safe `richtext` **data brick** (a leaf) so an agent can
  author a block of text with **inline mixed formatting + inline links** — a
  paragraph where "this **word** is bold" and "[this](link) is a link" flow in one
  wrapping line. The single-string `text` brick (one value, one style) cannot
  express this. This unblocks real prose UI (articles, messages, formatted copy)
  and is a surviving native brick in the locked model.

## Current Problem

- **Who:** the **agent author**. Today the only text primitive is `text` — one
  string with one style. Any mixed-format sentence (a bold word, an inline link, a
  quote, a bulleted list) is impossible; faking it with a row of separate `text`
  nodes breaks natural wrapping and reads as clumsy boxes. So agents cannot render
  ordinary formatted prose, which is a large fraction of real UI copy.

## User Scenario

### Who
- The **agent author** emits a `richtext` node; the **renderer** flows its blocks
  and runs with mixed formatting; the **end-user** reads formatted prose and
  clicks inline links (which resolve like any Facet action / navigate).

### Trigger
- The agent emits a `richtext` node in the tree (or patches one in).

### Steps
1. The agent authors `richtext` with `blocks[]` (a FLAT list): each block is
   `{ type: paragraph|heading|listItem|quote, level?, depth?, runs: Run[] }`.
2. Each `Run` = `{ text, marks?: Mark[] }`, where `Mark` is a **closed, extensible
   tagged union**: `{kind:"bold"} | {kind:"italic"} | {kind:"underline"} |
   {kind:"strike"} | {kind:"code"} | {kind:"link", target}`. The renderer flows
   the runs as one block, wrapping naturally, applying each mark's theme-owned look.
3. A `link` mark's `target` is either an internal Facet action (`navigate`/`agent`,
   reusing `FacetAction`) OR a **gated external URL** (a safe-href, mirroring
   `media.src`'s `isSafeMediaSrc`). Clicking resolves like any action; an external
   URL just navigates the browser (no fetch).
4. `listItem` blocks carry a bounded `depth` for nested bullets; `heading` blocks a
   bounded `level` (1–3). The renderer maps `level`/`depth`/mark kinds to the theme.

## Input/Output Examples

### Example 1 (Normal — mixed formatting + inline link)
- Input: `{ type:"richtext", blocks:[ { type:"paragraph", runs:[
  {text:"Welcome to "}, {text:"Facet", marks:[{kind:"bold"}]},
  {text:", read the "}, {text:"docs", marks:[{kind:"link", target:{kind:"navigate", to:"docs"}}]},
  {text:"."} ] } ] }`.
- Output: one wrapping paragraph — "Welcome to **Facet**, read the docs." with
  "Facet" bold and "docs" a clickable link that navigates locally.

### Example 2 (Edge — unknown mark / block, malformed run)
- Input: a run with `marks:[{kind:"blink"}]` (unknown), a block `type:"callout"`
  (unknown), a run missing `text`, and a `link` mark with `target:{url:"javascript:alert(1)"}`.
- Output: `blink` dropped (the run's text still renders plain), `callout` rendered
  as a paragraph, the malformed run skipped, the unsafe URL dropped (link renders
  as plain text or inert). Nothing throws; a richtext with zero valid runs → empty.

## Constraints (What Not to Do)

- **No HTML / no markdown parsing / no DSL (invariant #2).** The agent authors the
  closed structured shape (blocks/runs/marks) directly — Facet never parses an
  HTML or markdown string, never evaluates an expression. marks, block types, and
  link kinds are each a CLOSED set.
- **Leaf brick — no child NODES.** `blocks`/`runs` are the brick's own DATA (not
  tree node ids). richtext holds no `children` and is NOT `from`-bound (a store
  cell is one scalar; richtext is nested structured content → always inline).
- **Markdown-complete ceiling, not a document editor.** FLAT `blocks[]` + a list
  `depth` reaches Markdown-grade display (paragraph/heading/nested-list/quote/…).
  A true TREE of nested blocks (a block containing sub-blocks — Notion-style) is
  OUT of scope: that is document *editing*, and editing rich text is the deliberate
  boundary (display is in scope, editing is not).
- **v1 marks are on/off + link; VALUED marks (highlight/color) are deferred** — but
  the `Mark` union shape must already allow a mark to carry attrs (so highlight/
  color add as union members later without a breaking reshape).
- **v1 lists are unordered (bullets) + depth; ordered lists deferred.** **No code
  BLOCK** (multi-line) in v1 — only the inline `code` mark. Both add later as new
  closed members.
- **The external-URL link is the ONLY new trust surface** — it must be a gated
  safe-href (browser navigation), never a fetch, never an arbitrary scheme.

## Policy & Edge Cases

| Scenario Step | Failure Policy | Boundary/Validation | Concurrency |
|---|---|---|---|
| author blocks | unknown block `type` → render as `paragraph` (degrade), never drop the text | closed `BLOCK_TYPES`; blocks/runs arrays capped | pure render |
| author marks | unknown mark `kind` → drop the mark, keep the run text | closed `Mark` union; marks array capped per run | pure render |
| run text | missing/non-string `text` → skip the run; zero valid runs → empty richtext | text length-capped (reuse MAX_NODE_BODY_CHARS) | pure render |
| link mark | external URL failing safe-href → drop the link (render plain/inert); internal target validated as `FacetAction` | gated `isSafeHref` (mirror `isSafeMediaSrc`); closed link kinds | a link click resolves like any action (navigate local / agent event) |
| heading/list | `level` outside 1–3 / `depth` over cap → clamped | bounded `level` + `depth` | pure render |

## Done Criteria (Testable)

- [ ] **DC-001 (happy render):** a `richtext` with mixed marks + an internal link
  renders as ONE flowing, wrapping block with the correct per-run emphasis and a
  clickable link that resolves the action. → Verify: `@facet/react` jsdom render
  test.
- [ ] **DC-002 (fail-safe):** unknown mark kind → dropped (text kept); unknown block
  type → paragraph; malformed/missing-text run → skipped; zero valid runs → empty;
  never throws. → Verify: core validation + react fail-safe tests.
- [ ] **DC-003 (closed / no DSL):** validation accepts ONLY the closed mark kinds,
  block types, and link kinds, dropping anything else with an issue; no HTML/
  markdown string is parsed anywhere. → Verify: `@facet/core` validate tests +
  grep (no parser).
- [ ] **DC-004 (link safety, boundary):** an external-URL link is gated by safe-href
  (a `javascript:`/`data:`/disallowed-scheme URL is dropped); an internal
  `navigate`/`agent` link reuses `FacetAction`; no client fetch is introduced. →
  Verify: core safe-href tests + a react link-render test.
- [ ] **DC-005 (leaf / not store-bound):** richtext holds its own blocks/runs (no
  child node ids, no `from`); it is sanitized inside `validateTree`; nested
  blocks/runs/marks are bounded. → Verify: validate + tree tests.
- [ ] **DC-006 (extensible + additive):** `Mark` is a closed EXTENSIBLE tagged union
  (a future/unknown kind degrades, DC-002); adding `richtext` is additive (existing
  trees byte-identical); richtext composes with the PR-1.5 `Styleable` pack. →
  Verify: existing suites stay green + an unknown-kind test.
- [ ] **DC-007 (list depth):** `listItem` blocks with a bounded `depth` render nested
  bullets; over-cap depth clamped, never throws. → Verify: react render test.

## Invariant Fit (Facet-specific — REQUIRED)

| # | Invariant | Status | Note / mitigation |
|---|---|---|---|
| 1 | Responsibility boundary | OK | Display content; an external link navigates the browser, it is not a fetch/data-source. No backend affordance. |
| 2 | Mechanism vs policy (no DSL) | TOUCHES (mitigation) | Closed `Mark` union + closed block types + closed link kinds; the agent authors the structured shape, never HTML/markdown/expression. Facet parses nothing. |
| 3 | Fail-safe | TOUCHES (mitigation) | Unknown mark/block → degrade to plain; malformed run skipped; unsafe URL dropped; never throws. |
| 4 | Declarative + tokens only | OK | Marks are semantic NAMES the theme renders (bold→theme weight, code→theme mono); block style via the token `Styleable` pack; no raw CSS/pixels. |
| 5 | Flow-only safety | OK | Runs flow inline and wrap within the block; blocks stack; no overlap/positioning. |
| 6 | Two-writers | OK | Pure content; a link click is a `navigate`/`agent` action with the existing semantics (no new local writer). |
| 7 | Backend via agent, never client fetch | TOUCHES (mitigation) | The external-URL link is the one outside-touching surface — a GATED safe-href that only navigates the browser (never fetches); internal links reuse `FacetAction`. No client fetch/resolver added. |
