# Feature Intake Brief: PR-5b — Container Components as Composition References

> Generated by `/feature-intake`. Plain-language, testable, invariant-checked.

## Goal

- Remove `card`, `section`, and `emptyState` as dedicated component node types while preserving their useful UI patterns as native `box`/`text`/`button` composition references.
- Make the removal an atomic hard cutover so every built-in asset, seed, renderer, agent-facing contract, and document agrees on the smaller node vocabulary.

## Current Problem

- `card` and `section` mainly package structure and styling that native `box` and `text` nodes can already express.
- `emptyState` packages structure plus an action that the agent can author directly as a child `button` with `onPress`.
- Keeping these shapes as both node types and composition examples duplicates concepts and makes the closed node vocabulary larger than necessary.
- Removing only the public node types would invalidate built-in compositions and quickstart seeds that still contain the retired nodes.

## User Scenario

### Who

- An agent author using Facet and an end-user viewing the rendered page.

### Trigger

- The agent needs a card, a general content section, or an empty-state presentation after PR-5b lands.

### Steps

1. The agent reads or retrieves a `card`, `section`, or `empty-state` composition reference.
2. The reference shows the pattern using only native `box`, `text`, and, for an actionable empty state, `button` nodes.
3. The agent authors those explicit native nodes and supplies any copy and action itself.
4. Facet validates and renders the tree without requiring a dedicated `card`, `section`, or `emptyState` node type.

## Input/Output Examples

### Example 1 (Normal)

- Input: The agent authors a section-shaped `box` containing eyebrow, title, body, and content nodes, using token names for layout and appearance.
- Output: The content renders with the intended section presentation, and the stage tree contains no `section` node.

### Example 2 (Normal — Empty State Action)

- Input: The agent authors an empty-state `box` with title and body `text` children plus a `button` child containing `onPress`.
- Output: The page shows the empty state and the button reports the authored action through the existing action path.

### Example 3 (Edge/Error)

- Input: An old or malformed tree contains `{ "type": "card" }`, `{ "type": "section" }`, or `{ "type": "emptyState" }`.
- Output: Core validation rejects it, agent tools return a clean type error, and the React renderer skips a stale raw node without throwing.

## Constraints (What Not to Do)

- Do not retain aliases, compatibility parsers, migration shims, or deprecated public node types; this is an unreleased breaking hard cutover.
- Do not add a new ability for `box` to inherit or “wear” recipes belonging to removed component types. Use existing `box` behavior and inline token references.
- Do not emit raw HTML, JavaScript, CSS values, or pixel scalars; styles remain declarative token names.
- Do not add slots or automatic action wiring. The LLM authors composition children, copy, and `button.onPress` explicitly.
- Do not change patch transport, runtime action semantics, backend behavior, or local action execution.
- Do not touch the `stat` component; that work belongs to PR-6.
- Accept that replacing the specialized renderers with native boxes removes their implicit `<section>`, `<h2>`, and `<h3>` output. PR-5b does not add a replacement heading/landmark feature.
- Do not merge or push without explicit owner approval, and do not commit implementation directly on `main`.

## Policy & Edge Cases

| Scenario Step | Failure Policy | Boundary/Validation | Concurrency / Duplicate Trigger |
|---|---|---|---|
| Author a native composition | Reject any node or style value outside core's closed vocabulary | Only `box`, `text`, `button`, and other surviving core nodes/tokens are accepted | Existing stage patch ordering remains unchanged |
| Author an empty-state action | Require the normal validated `button.onPress` shape | No implicit action, action label, or slot is synthesized | Existing duplicate-action and two-writer policies remain unchanged |
| Submit a retired type through `set_node` | Return a clean type-validation error; never execute or partially insert it | `asNode` must confirm the type is an own property of the executor table | Repeated invalid calls fail independently and do not mutate the stage |
| Render a stale raw retired node | Skip the node and continue rendering valid siblings | Renderer has no registry entry for the retired type | Re-renders remain deterministic and do not throw |
| Load built-in compositions and seeds | All shipped definitions must validate after the cutover | Rewrite every retired internal node and stale parent hint atomically | Static assets have no concurrent writer |

- [ ] DC-002 (Fail-safe boundary): validation rejects all three retired types while the React renderer blank-degrades stale raw instances and renders valid siblings without throwing. → Verify: core validation tests and React renderer tests.

- [ ] DC-011 (Atomic consistency): no shipping code, asset, seed, or documentation treats the three names as node types; remaining occurrences are limited to deliberate negative/fail-safe tests or composition reference names. → Verify: repository-wide `git grep` with occurrence classification.
