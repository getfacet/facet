# Feature Intake Brief: Quickstart Built-in Agent (Bundle A)

> Generated by `/feature-intake`. Plain-language, testable, invariant-checked.
> Three parts, one flow: field value wiring â†’ built-in reference LLM agent
> (`@facet/quickstart`) â†’ live-link hard gate.

## Goal

- A new user goes from `pnpm add`/`npx` to **touching a live, agent-drawn page
  in a browser** with nothing but an LLM provider API key and (optionally) one
  markdown file describing what the page is for.
- The same runnable stack becomes the project's own E2E test harness: every
  future work item is verified against a real server + real link, not only
  unit tests (a new hard gate joins `/verify` â†’ `/code-review`).
- Forms work end-to-end: an agent can draw fields + a submit button and
  actually receive what the visitor typed.

## Current Problem

- **Field values never reach the agent.** The renderer draws `<input>`s, but
  an `agent` action carries only its static, agent-authored `payload`. Any
  form scenario (contact, signup, search) silently does nothing with the
  visitor's input. Affected: every agent author; every end-user who types
  into a Facet page today.
- **No zero-setup first-run.** The three brain-jacks (in-process / dial-in /
  CLI+bridge) all require assembly; the only "just works" path (`playground
  serve`) spawns a local `claude` CLI, lives in an unpublished app, and can't
  be `npm install`ed. Affected: every prospective adopter (the first five
  minutes decide adoption).
- **No live verification.** The DoD chain is unit-level (`/verify`) and
  review-level (`/code-review`); nothing boots the real server and drives the
  real link, so transport/renderer regressions can pass the gates. Affected:
  us (maintainers), every future bundle (B/D lands on top of this).

## User Scenario

### Who
- **S1 (end-to-end quickstart):** a prospective adopter (agent author) + the
  end-user visiting the page they stand up.
- **S2 (forms):** an agent author whose agent draws a form; the end-user who
  fills it.
- **S3 (gate):** us, the maintainers, running the DoD chain.

### Trigger
- S1: `OPENAI_API_KEY=â€¦ npx facet-quickstart --guide ./my-page.md`
- S2: the end-user presses a submit box whose action declares `collect`.
- S3: a work item reaches the hard-gate step.

### Steps (S1 â€” quickstart)
1. User sets a provider key env var and runs the one command.
2. Facet: starts the local server with the built-in agent wired in, prints the
   link (e.g. `http://localhost:5292`), and states which provider/model it is
   using (default: OpenAI; `ANTHROPIC_API_KEY` alone â†’ Anthropic).
3. User opens the link; a `visit` event flows; the built-in agent composes the
   3-layer prompt (â‘  Facet usage guide seeded from `STAGE_SPEC`, â‘ˇ the guide
   file, â‘˘ live context: event + current stage + chat history) and calls the
   provider; the returned brick tree renders.
4. User chats in the dock; the agent patches the stage live. Expected result:
   a personal, evolving page with zero code written.

### Steps (S2 â€” forms)
1. The agent draws `field`s inside a box and a pressable box with
   `onPress: { kind: "agent", name: "submit", collect: "<form-box-id>" }`.
2. End-user types into the fields and presses the box.
3. Renderer: gathers the current values of every `field` in the `collect`
   box's subtree into `fields: { <field.name>: <value> }` and emits the
   action event carrying both the static `payload` and the collected
   `fields`.
4. The agent receives the event and acts (e.g. replies using the values).

### Steps (S3 â€” gate)
1. A work item finishes; the live gate runs (skill, e.g. `/live-test`).
2. Tier 1 (always, blocking): server boots with the **deterministic stub
   brain** (no key, fixed responses); a scripted client visits, chats, and
   presses a collect button; assertions check the full path (SSE snapshot,
   patch application, field delivery).
3. Tier 2 (blocking only when the change touches `@facet/quickstart`'s
   prompt/provider/parsing code, and a key is present): one real-LLM smoke
   turn with the default provider; **loose assertions** (a valid tree
   rendered; no error), never exact-content matching.
4. Tier 3 (pre-merge/release): the smoke runs against both providers.

## Input/Output Examples

### Example 1 (Normal â€” S2 forms)
- Input: stage has `field(name:"email")` and `field(name:"plan")` inside box
  `signup`; end-user types `a@b.c`, `pro`, presses the box with
  `{kind:"agent", name:"submit", collect:"signup", payload:{src:"hero"}}`.
- Output: the agent receives an `action` event with `name:"submit"`,
  `payload:{src:"hero"}`, and `fields:{email:"a@b.c", plan:"pro"}`.

### Example 2 (Edge â€” unknown collect target)
- Input: `collect:"no-such-box"` (or the box exists but contains no fields).
- Output: the action event fires normally with `fields:{}`; nothing throws;
  the renderer stays up (fail-safe, same posture as unknown
  `navigate`/`toggle` targets).

### Example 3 (Edge â€” malformed LLM output)
- Input: the provider returns text that is not a valid tree/patch set.
- Output: the quickstart agent degrades: the stage is not corrupted (existing
  `validateTree` + per-op salvage), the visitor gets an apologetic chat line,
  the server stays up. Never a crash, never a broken page.

### Example 4 (Edge â€” missing key)
- Input: `npx facet-quickstart` with no provider env var set.
- Output: exits non-zero with a friendly message naming `OPENAI_API_KEY` /
  `ANTHROPIC_API_KEY` and pointing at `--stub` for a keyless look-around.

## Constraints (What Not to Do)

- **No new backend affordances.** The built-in agent has no tools in v1 â€” it
  draws UI and chats; it does not fetch/compute/store domain data. (No
  `kind:"fetch"`, no client-side network access from the page.)
- **No raw values anywhere.** `collect` is a node-id reference; collected
  values ride the existing event channel; no new HTML/CSS/scalar surface.
- **No key persistence.** Keys are read from env vars only â€” never written to
  disk, never echoed to logs, never sent anywhere but the provider.
- **Out of scope for this bundle:** themes/kits as data (Bundle B), new
  interaction kinds (Bundle D), `.env` loading, interactive key prompts,
  providers beyond OpenAI + Anthropic, guide hot-reload (restart to re-read),
  streaming token-by-token render (turns are atomic in v1), multi-visitor
  scale hardening (reference-server trust model unchanged).
- The three existing brain-jacks are NOT removed or changed â€” only re-ranked
  in docs as "advanced".

## Policy & Edge Cases

| Scenario Step | Failure Policy | Boundary/Validation | Concurrency / Duplicate Trigger |
|---|---|---|---|
| S1-1 run command | missing key â†’ non-zero exit + guidance (Ex.4); port busy â†’ clear error naming the port + `--port` | key format not validated locally (provider rejects); guide path missing â†’ non-zero exit naming the path | second instance on same port fails fast, no half-started state |
| S1-3 first paint | provider error/timeout â†’ visitor sees fail-safe page shell + error chat line; server stays up | provider timeout bounded (default 60s/turn); response size bounded before parse | multiple visitors = independent sessions (existing per-visitor serialization) |
| S1-4 chat turn | malformed LLM output â†’ Ex.3 degrade; provider 4xx/5xx â†’ error say, stage untouched | chat history in layer â‘˘ capped (last N turns, default 20) to bound prompt size | rapid messages queue per visitor (existing serial queue); no interleaved turns |
| S2-3 collect on press | unknown/empty target â†’ `fields:{}` (Ex.2) | per-value length cap (default 2000 chars, truncate); only `field` nodes in the subtree contribute; non-string values coerced to string | double-press sends two events (agent's problem to dedupe, as today); collection is a synchronous snapshot at press time |
| S3-2 stub gate | any assertion fails â†’ gate FAIL (blocking) | stub responses are fixtures checked into the repo | gate boots on an ephemeral port; parallel runs don't collide |
| S3-3 smoke gate | provider unreachable â†’ gate reports SKIPPED-with-reason (not PASS); if the change touched quickstart, SKIPPED = FAIL | loose assertions only (valid tree, no error) | one smoke turn; no retries beyond one to keep cost bounded |
